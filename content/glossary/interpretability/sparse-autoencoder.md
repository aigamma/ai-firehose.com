A sparse autoencoder, or SAE, is a small network trained to reconstruct the internal activations of a larger model while passing them through a wide hidden layer that is forced to stay mostly silent. The hidden layer has many more units than the activation it is reading, often by a factor of eight, sixteen, or more, and a sparsity penalty pushes all but a handful of those units to zero on any given input. The result is a learned dictionary: each hidden unit corresponds to a direction in the original activation space, and the model's representation at that layer is re-expressed as a sparse sum of these dictionary directions, called features.

The SAE has become the central workhorse of modern mechanistic interpretability because it is the most effective known answer to superposition. A network stores more features than it has dimensions by overlapping them as non-orthogonal directions, which is why individual neurons are polysemantic and hard to read. By learning an overcomplete basis under a sparsity constraint, the SAE undoes that compression: it recovers the underlying directions so that each one tends to fire for a single human-recognizable concept. Features extracted this way are far more monosemantic than raw neurons, which makes them tractable units for building a circuit account of the model.

The method is a form of dictionary learning, and its discipline lies in the tension between two terms. The reconstruction loss demands that the sparse code preserve the original activation faithfully, so that nothing the network actually computes is thrown away. The sparsity penalty, classically an L1 term on the hidden activations, demands that as few features as possible be used at once, which is what forces each feature to specialize. Tuning this trade-off is the core engineering problem, and variants such as the top-k SAE, which fixes the number of active features directly, and the gated and JumpReLU designs, which sharpen the on-off decision, were introduced to reduce the bias that a naive L1 penalty introduces.

A trained SAE supports the whole interpretability pipeline. Each feature can be named by feature visualization, by collecting the inputs that activate it most strongly, and its causal role can be tested by clamping it on or off, which connects directly to activation steering. Large-scale efforts have used SAEs to extract millions of features from production models and to map features that correspond to concrete and even safety-relevant concepts. The approach is not finished: SAEs leave reconstruction error, can split one concept across several features or absorb several concepts into one, and offer no guarantee that the dictionary they find is the one the network truly uses. Even so, the sparse autoencoder is currently the most scalable bridge from raw, superposed activations to features a person can understand and intervene on.
