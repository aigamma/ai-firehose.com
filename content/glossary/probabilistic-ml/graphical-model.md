A graphical model is a marriage of probability theory and graph theory that represents a joint distribution over many random variables as a graph. Each node is a random variable and the edges encode how the variables depend on one another. The real content is in the missing edges: their absence asserts conditional independence, the statement that once you know certain variables, others carry no further information about each other. By exploiting that structure, a graphical model factorizes an enormous joint distribution into a product of small local pieces, turning a table that would be astronomically large into something compact and computable.

Graphical models matter because they give probabilistic modeling a language and a set of reusable algorithms. They come in two main flavors. Directed graphical models, also called Bayesian networks, use arrows to express how each variable is generated from its parents, encoding a causal-style story and a clean application of [Bayes' theorem](bayes-theorem). Undirected graphical models, or Markov random fields, use plain edges to express mutual compatibility without a direction of causation, which suits images and spatial data where neighboring pixels simply tend to agree. Both reduce to the same core question once built: given evidence on some variables, what is the distribution over the rest.

The mechanics rest on the factorization the graph licenses. In a directed model the joint distribution is the product over all nodes of each variable's [conditional probability](conditional-probability) given its parents, so specifying the model means specifying these small conditional distributions rather than the full joint. Answering a query, computing the posterior over hidden variables given observed ones, is called inference, and for graphs without loops it can be done exactly and efficiently by message passing, which propagates local beliefs along the edges. For densely connected graphs exact inference becomes intractable, which is precisely where the approximate methods of probabilistic machine learning enter: [variational inference](variational-inference) and [Markov chain Monte Carlo](markov-chain-monte-carlo) are the standard tools for inference in graphical models too complex to solve exactly.

Graphical models are the unifying backbone under many specific techniques. A [hidden Markov model](hidden-markov-model) is a graphical model with a particular chain structure; a Gaussian mixture, a Kalman filter, and a topic model are each graphical models with their own characteristic shape. Viewed this way, much of probabilistic machine learning is the study of how to design the graph for a problem, how to learn the local distributions from data, and how to run inference once the structure is fixed. The graph makes the modeling assumptions explicit and visible, which is one reason graphical models remain a clarifying way to think even about the [latent variable models](latent-variable-model) that power modern deep generative systems.
