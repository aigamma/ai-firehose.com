Quasi-Newton methods occupy the middle ground between gradient descent, which uses only first-derivative information and converges slowly near a minimum, and Newton's method, which uses the full Hessian for fast convergence but pays an enormous price to form and invert it every step. The quasi-Newton idea is to enjoy Newton-like convergence while never touching the actual Hessian. Instead of computing second derivatives, these methods build up an estimate of the curvature, or directly of its inverse, from the only second-order signal that is free to observe: how the gradient changes as the parameters change.

The engine is the secant condition. As the optimizer moves from one point to the next, it sees both the step it took and the resulting change in the gradient. For a quadratic function the Hessian is exactly the matrix that maps the step to the gradient change, so any honest curvature estimate ought to reproduce that relationship for the most recent step. A quasi-Newton method enforces this condition on its running estimate and updates the estimate with the smallest change consistent with it, so each iteration the approximation absorbs one more curve of evidence. The celebrated BFGS update, named for Broyden, Fletcher, Goldfarb, and Shanno, does this while keeping the estimate symmetric and positive definite, which guarantees the search direction always points downhill, a property Newton's method lacks far from a minimum.

This matters because it changes the cost structure of second-order optimization. Newton's method needs the full Hessian, quadratic in the number of parameters to store and cubic to invert, which is hopeless for large problems. Quasi-Newton methods maintain their approximation with linear-algebra updates and, by tracking the inverse directly, compute the search direction with a matrix-vector product rather than a matrix inversion, removing the per-step cubic cost entirely. They converge superlinearly, faster than the linear rate of gradient descent and approaching the quadratic rate of Newton's method as the approximation sharpens, while using only gradients. Each step is paired with a line search that satisfies conditions ensuring the curvature update stays well behaved.

The remaining bottleneck is memory: a dense inverse-Hessian approximation is still quadratic in the parameter count to store, which is why classic BFGS is reserved for problems up to thousands of variables, the regime of many scientific and statistical fitting tasks. For the millions or billions of parameters in deep networks, the limited-memory variant L-BFGS keeps only the last few step-and-gradient pairs and reconstructs the action of the inverse Hessian on demand, trading some accuracy for a linear footprint. Quasi-Newton methods sit alongside conjugate gradient and trust-region methods in the toolkit of optimizers that exploit curvature without forming it explicitly, and BFGS in particular is the default behind many off-the-shelf nonlinear optimizers.
