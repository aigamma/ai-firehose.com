The problem the LSTM was built to solve is not memory, it is multiplication. A plain recurrent neural network revisits its hidden state at every step by multiplying through a weight matrix and a nonlinearity, and when backpropagation through time runs that chain in reverse, the gradient is multiplied by those same factors over and over. Across a long sequence the product collapses toward zero, the vanishing gradient, and the network simply cannot learn that something early mattered to something late. You can make the recurrence remember in principle and still find it untrainable in practice.

Hochreiter and Schmidhuber's 1997 answer was to build a path through time where information is added rather than multiplied. Alongside the ordinary hidden state, the LSTM carries a separate cell state, and the cell state moves from one step to the next mostly by addition, with only light linear interaction. Think of it as a conveyor belt running the length of the sequence: the network sets things on it and takes things off, but the belt itself does not pass the contents through a fresh nonlinearity at every step. Because the backward path along that belt is near-additive, the gradient can travel tens or hundreds of steps without being crushed. The original work called this the constant error carousel, and it is the real invention; the gates are the controls bolted onto it.

Those gates are what turn a passive conveyor belt into a memory the network can program. Each gate is a small layer with a sigmoid activation function squashing its output between zero and one, so it acts as a soft, differentiable valve rather than a hard switch. The forget gate sets how much of the current cell state survives to the next step, the input gate sets how much new candidate content gets written, and the output gate sets how much of the cell state is exposed as the visible hidden state. Nothing tells these gates what to gate; the network learns when to hold, when to overwrite, and when to reveal, the whole apparatus trained end to end. The deeper move is that "remember this for later" has been turned from a discrete decision, which a gradient cannot flow through, into a continuous one that it can.

For nearly two decades this was the default way to model sequences, behind machine translation, speech recognition, and text generation, with the slightly leaner gated recurrent unit folding the cell and hidden states back together. The attention mechanism and the transformer have since displaced it, for a reason that is the mirror image of the LSTM's strength: an LSTM still walks the sequence one step at a time, so it cannot exploit a modern accelerator's parallelism the way attention's all-at-once comparison can, and a fixed-size cell state is a tight bottleneck for very long contexts. The idea it proved is what survived. Engineer a route along which the gradient is preserved instead of repeatedly attenuated, and depth in time stops being the enemy, a lesson the residual connection would later reuse for depth across layers.
