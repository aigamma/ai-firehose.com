Rejection sampling fine-tuning is a simple and surprisingly effective alternative to reinforcement learning for improving a model after pretraining. The recipe has three moves repeated in a loop: sample, filter, fine-tune. For each prompt the current model generates many candidate responses, a scorer keeps only the good ones, and the model is then fine-tuned on that curated set with ordinary supervised learning. Because the surviving examples are the model's own best work, the model is in effect taught to imitate the top slice of its own output distribution, ratcheting its typical behavior upward toward its occasional best behavior.

The filtering step is where the quality signal enters, and there are two common forms. When the task has a checkable answer, such as a math problem with a known solution or code that must pass tests, a verifier accepts only responses that are actually correct, so the fine-tuning set is guaranteed clean; when the task is open-ended, a reward model scores the candidates and the highest-scoring ones are kept, the best-of-N idea used as a data generator rather than only at inference time. Either way the model never trains on its mediocre samples; rejection refers exactly to discarding everything below the bar.

This method matters because it captures much of the benefit of reinforcement learning from human feedback while staying inside the familiar, stable machinery of supervised fine-tuning, the keeper. There is no policy gradient, no value function, and no delicate reinforcement learning loop to stabilize, so it is far easier to implement and debug, and it parallelizes cleanly since sampling and scoring are embarrassingly parallel. It was used as a stage in the post-training of several major model families, often as a warm start before a final reinforcement learning pass or as a standalone improvement loop, and the data it produces is a form of synthetic data, the procedure closely related to expert iteration in reinforcement learning and to self-distillation, where a model is distilled from a filtered version of itself.

The limitations follow from the same self-referential design. The model can only learn from behaviors it already produces at least occasionally, so rejection sampling fine-tuning sharpens and reliably surfaces latent skills but rarely invents genuinely new ones, and its ceiling is set by the quality of the scorer: a weak reward model will pass flawed responses and a biased one will entrench its bias, the same reward-hacking pressure that haunts RLHF, here expressed as a slow drift in the fine-tuning data. Diversity can also collapse over many rounds as the model concentrates on a narrow set of high-scoring patterns, so practitioners watch for this and refresh prompts or mix in fresh demonstrations.
