## Session 9, 2026-06-03 (overnight autonomous loop): CI gates for the doc harness and the knowledge graph

- **Two CI gates now guard the harness against rot, built on one reusable pattern.** A checker that judges against the COMMITTED tree (`git ls-files`, never the local disk, so local and CI cannot diverge), wrapped in a `node:test` so `npm test` blocks new drift, with a baseline ratchet for pre-existing debt. `scripts/check_docs_fresh.mjs` does this for backtick paths and `npm run` scripts named in the current-state docs (the append-only history logs are excluded by design); `scripts/check_glossary.mjs` does it for glossary `related:` cross-links. How to apply: when a class of reference can silently go stale, write a committed-tree checker plus test rather than trusting review to catch it.

- **A silent drop is rot; make the build fail loudly instead of tolerating bad input.** `build_glossary` silently dropped any `related:` slug that did not resolve (its `.filter((r) => r.label)` on the related mesh), hiding broken cross-links. The new gate surfaced 26 of them, and the standout was that `backpropagation`, the single most foundational training algorithm, was missing entirely despite 16 entries linking to it. How to apply: wherever a build tolerates malformed input by dropping it, add a check that fails on it instead. Silent tolerance hides real gaps, and the most-referenced missing thing is exactly what no single author notices.

- **The doc-staleness gate judges the committed or staged tree, so stage new files before running it.** A doc that references a file added in the same change flags as stale until that file is staged (staged counts as tracked in `git ls-files`). How to apply: `git add` the new files first, then run the gate, then commit. This is a workflow order, not a bug.

- **This repo can have more than one agent committing in parallel; regenerate artifacts defensively.** During this loop a concurrent contributor added hub images, a Pinecone embedding of the glossary, and `docs/GLOSSARY.md`. A transient doc-gate failure turned out to be a race (a doc referenced `worker/pipeline/embed_glossary.mjs` in the window before that file's commit landed) and self-resolved on re-check. `build_glossary` preserved the concurrent image work because it reads `content/glossary/images.json` and re-applies it on every rebuild. How to apply: before committing regenerated artifacts, diff them to confirm you are not clobbering another contributor's fields; lean on `content/` as the source of truth, since content-sourced additions survive any rebuild; and treat a one-shot gate failure as possibly a race, re-checking before you "fix" it.

- **Overnight loop discipline: keep working in-turn; do not ScheduleWakeup between every unit.** ScheduleWakeup is a failsafe so the loop survives a turn ending, not a per-unit checkpoint. Pausing after each commit burns wall-clock and a prompt-cache miss for nothing. How to apply: chain the valid queued tasks within one turn, and let the turn end only on a hard blocker or real context pressure. (Reinforces the global pacing rule; recorded here because it recurred across the loop.)

- **Building a visual feature blind: verify everything except the pixels, against non-visual oracles.** The Atlas view (a 31-node SVG constellation in `Glossary.jsx`) was built with no way to see the render. Evidence-over-assurance still applies; the oracle just has to be something other than my eyes. The pattern that worked, in layers: (1) push the logic into a PURE, deterministic, unit-tested builder (`scripts/lib/atlas.mjs`) so the data the view consumes is provably correct before any pixel exists; (2) confirm the production build compiles the JSX; (3) replay the EXACT geometry the renderer will compute (the same `C + x*RAD` mapping, node radii, edge endpoints) over the real artifact in a throwaway node check, and assert every coordinate is in-bounds (no clipping) and every edge resolves to a real node. That proves the numbers the SVG draws are valid even though I cannot see the SVG. (4) Design for blind robustness: a fragile SVG (in-SVG text layout overflows and overlaps unpredictably at different sizes) was paired with a readable, clickable HTML legend that is both the accessibility path and the interaction, so even an imperfect constellation degrades to a usable control. How to apply: when you cannot see the output, find the non-LLM, non-visual oracle (compile, replay-the-geometry, bounds asserts), commit what those oracles verify, and state plainly in the commit which single thing (here, the aesthetic) still needs a human eye. Do not assert visual quality you did not verify.

## Session 8, 2026-06-03: Reality is the test; the torch-pass to an editorial intelligence service

- **Evidence over assurance: cash out every load-bearing claim against a narrator-independent oracle.** A redirect deploy on a sibling project reported success (`locked: true`, a matching published-deploy id) while the live URL served `200`, not the intended `302`, because `netlify deploy --dir <bundle>` reads `netlify.toml` from the CURRENT working directory, not from `--dir`, so it applied THIS repo's SPA catch-all `/* -> /index.html 200` and shadowed the bundle's own redirect. It was caught only by `curl -sSI` against the live URL. How to apply: a tool's "success" message and a matching id are narrative; the HTTP status line, the build exit code, the file's actual bytes are evidence. Verify before reporting done. Deploy a standalone redirect or static bundle by `cd`-ing into it first. This rule now lives in the global `~/.claude/CLAUDE.md` ("Evidence Over Assurance"), so it applies across every project.

- **The Opus-for-enduring-content rule is now wired into the pipeline, not just subagent authoring.** `MODELS` (`worker/lib/anthropic.mjs`) gained an `enduring` tier (`claude-opus-4-8`); the daily briefing (`briefing.mjs`) moved off Sonnet and the glossary definitions (`define.mjs`) moved off Haiku, both onto Opus 4.8. Classification stays on Haiku. How to apply: the durable, low-volume prose a smart reader keeps gets the strongest model; the high-volume churn gets the cheap one. (Operationalizes Session 7's authoring rule into config.)

- **The rotation question came back, and was re-confirmed dropped from the data, again.** Re-pulled the pre-removal trails: most concepts pinned to the `[55,145]` clamp rails on attention counts of 5 to 11, with single-step jumps of 45 to 62 on a 90-wide scale, exactly Session 6's post-mortem. A 5 to 7 point tail or smoothing cannot round data that is rail-pegged or static; it would only hide the jaggedness or invent motion. Stays gone. How to apply: when a removed feature is proposed for revival, re-check the original failure against current data before rebuilding; do not let a beautiful reference image (aigamma's dense daily price series) override the data regime you actually have (sparse, bursty counts).

- **North star sharpened: a personal intelligence service, editorial not instrumental.** Home was reframed from a dashboard (a flat tagline over a wall of boards introduced by a four-sentence instrument-manual paragraph) into an editorial front page: a serif masthead and standfirst, a dateline, and the Opus brief as the lead story, with the boards demoted to "What Moved" and the mono instrument-panel motif pulled back. The promise in one line: we structure the firehose so the reader takes what they want and leaves the rest, bottled just for them. How to apply: the test for any surface is whether it does the analyst's work FOR the reader (editorial) or makes the reader do it (instrumental); prefer the former. Voice: useful and literate, never cheesy.

## Session 7, 2026-06-03: The durable knowledge layer, wiki-style auto-linking, and parallel Opus authoring

- **The glossary now has two layers with opposite lifecycles, and the split is the architecture.** A DURABLE, authored knowledge base (`content/glossary/<category>/<slug>.md`, Markdown plus frontmatter) is compiled by `scripts/build_glossary.mjs` into the served hubs and index, marked `durable: true`, and MERGED non-destructively with the corpus-derived trending taxonomy: the authored definition, body, and related mesh overlay the corpus attention, rotation, neighbors, and items where a slug appears in both. The durable layer never expires; the corpus turns over each quarter. How to apply: when a system mixes permanent and ephemeral data, make the permanent layer a committed source of truth (`content/`) that a build re-merges, not a row in the same store the prune touches. It is self-healing: because the build runs as a `prebuild` AND at the end of the worker's glossary step (`run.mjs`), a corpus rebuild can never permanently drop the authored entries.

- **Width via parallel Opus subagents: two fan-out waves (19 then 12 category agents) authored 356 entries across 31 categories.** Wave 1 covered the core field; wave 2 went wider and more exotic (advanced mathematics like measure theory and optimal transport, deep learning theory, reinforcement learning, probabilistic ML, graph and geometric learning, systems, evaluation, cognitive science, and an advanced philosophy bridge to worldthought). Each agent owned a category and wrote Markdown files directly, pointing at the committed README spec plus three canonical example entries so independent agents produced a consistent format. How to apply: for a large authoring backlog, lock the format with a committed spec and canonical examples FIRST, then fan out one agent per partition writing files directly (not returning bloated payloads); a tolerant build parser (skip-and-warn on a malformed file) absorbs the variance. Subagents inherit the session model, so the durable content got Opus authorship, which is the rule: permanent content gets the strongest model, transient summaries get the cheap ones.

- **Build the rails while the fan-out runs.** The 19 authoring agents ran in the background while the main thread built the compiler, the pure matcher engine, the renderer, and the page integrations against the three seed entries. By the time authoring finished, the rails were done and one authoritative build wired it together. How to apply: a background fan-out plus main-thread rail-building overlaps the two largest time sinks; seed a few real examples so the rails are testable before the volume lands.

- **Separate pure logic from JSX so the wiki-linker is unit-testable without a React harness.** `src/lib/richtext.js` (buildMatcher, parseInline) is pure and tested with `node --test`; `src/components/RichText.jsx` only maps the token tree to elements. The matcher compiles every concept label and alias into one boundary-anchored, longest-first regex, fetched and built once for the whole app via `src/lib/useGlossary.js`. How to apply: when a component embeds real parsing or matching logic, lift it into a pure module the test runner can reach, and keep the JSX shell trivial.

- **Labels must beat aliases in the wiki-link matcher, or a concept links to the wrong hub.** The first pass registered labels and aliases together, first-wins by iteration order, so `attention-mechanism` listing `self-attention` as an alias shadowed the real `self-attention` concept and the link went to the wrong page. Fix: two passes, all labels first (authoritative), then aliases fill only unclaimed surface forms. How to apply: in any term-to-target matcher built from a mix of canonical names and synonyms, the canonical name takes precedence; a synonym never overrides another entity's real name.

## Session 6, 2026-06-02: The rotation plane was the wrong instrument; replace it with trend heat boards

- **A four-quadrant rotation plane (Mansfield, the prior art behind RRG) needs a persistent universe sampled continuously, and this corpus is the opposite, so the plane showed clamp artifacts, not rotation.** Pulled from the real `*_week.json`: 12 of 16 techniques had `ratio` pinned to exactly the `[55, 145]` clamp wall; three concepts that co-occur in the same items (`prompt-engineering`, `workflow-orchestration`, `agent-wrapper`) were byte-identical dots stacked on each other; and almost every trail had the same shape (appear at center, jump to a corner, slide down the wall), which is an exponential-decay kernel over a publish-date histogram, not a topic rotating. The cruelest part: the `pruneForPlane` "is it interesting?" gate keyed on distance from 100, but the clamp manufactures the MAXIMUM distance from 100 for the sparsest concepts, so the filter meant to drop the boring tail actively selected the clamp noise. Root cause: the series is a single ingest's `published_at` histogram, not longitudinal re-measurement; each item only ever contributes attention on its publish day, so even daily runs will not produce genuine rotation, they only add new dates. How to apply: before committing to a visualization, sanity-check it against the REAL artifact values, not the math in the abstract; a metric that looks principled (Mansfield normalization) can be degenerate on the data regime you actually have (sparse, bursty, recency-dominated, churning universe).

- **The replacement ranks by ABSOLUTE windowed growth (`delta`), and the metric had to be chosen against real numbers because the obvious normalized version also saturated.** First attempt was a symmetric normalized score `(recent - prior) / (recent + prior)` on the decayed level. Printing it on the real boards showed 12 of 16 pinned at exactly `+1.00`, the same saturation pathology as the clamp on the other axis, because the prior window is empty for any concept that first appeared inside the window (the common case here); worse, it ranked the clearest breakout (`test-time compute`, attention 9, breakout-flagged) at position 13, below twelve tiny brand-new concepts. The fix: rank by `delta = attention_now - attention_prev` in raw weighted-mention units (`windowTrend` in `attention.mjs`), which discriminates by the SIZE of the surge (a big breakout outranks a tiny one) and does not let a flat-but-large topic dominate (its now and prev are both high, so delta is near zero). The bounded `trend` is kept only for the up/steady/down arrow. The opinion board now shows genuine decliners at the bottom (`AI agents` delta -8.6, `test-time compute` delta -10.7: large but cooling), exactly the heating-versus-cooling read the plane never gave. How to apply: when "trending" is the goal on sparse data, prefer an absolute growth measure for the ranking (it degrades gracefully and never divides by a near-empty base); reserve any normalized ratio for display direction, where its saturation is harmless.

- **Removing a visualization is a footprint deletion, and the served artifact must stop carrying what nothing renders.** Swapping the two charts (`UnifiedRotationChart`, `RotationChart`) for the reusable `TrendBoard` orphaned `src/lib/rotationGeo.js` (and its test), the `ROTATION` and `TRAIL_PALETTE` registry constants, the `shape` field on `KINDS`, and the `trail` trajectory that `computeRotation` and `boards.mjs` wrote into every board entity. All were deleted in the same change, and the comments that named them (registry, `recompute_boards.mjs`, the clamp comment) were corrected, because a working tree where the docs lie is a process failure. The `ratio`, `momentum`, and `quadrant` values were KEPT: they still feed the concept hub Momentum card and the outlier z-scores, so they are not dead. How to apply: when cutting a feature, grep its whole footprint (component, shared geometry, registry constants, the data fields only it consumed, tests, doc references) and follow the dependency graph; distinguish genuinely-dead data (`trail`) from data with another live consumer (`momentum`).

- **The three written briefing columns (Top Movers, Breaking Out, Just Surfaced) were redundant once the boards rank by trend, so they folded into in-board badges.** A heat board sorted by `delta` with breakout and new flags shows movers (the top), breakouts (badged), and just-surfaced (badged) in one coherent view, so Home dropped the separate columns; the single breakout hero now reads from the live boards (`delta`), not the stale digest, so the lede always matches what the boards show. How to apply: when a new primary view subsumes several secondary lists, delete the lists rather than keeping both, since redundant surfaces drift.

- **Source breadth is cheap leverage for the briefing, and a background subagent verified every feed URL so no dead URL was committed.** Expanded `sources/blogs.json` from 7 to 22 feeds (frontier-lab blogs, heavy newsletters, tech press, policy and research-data orgs) and added three subreddits, each URL WebFetch-verified; labs with no public RSS (Anthropic, Meta AI, Mistral) were dropped rather than scraped or faked. The briefing only summarizes what the corpus ingested, so a wider, higher-authority, less educator-heavy diet directly improves the lede. How to apply: a tolerant RSS manifest plus a verify-before-add step is the cheapest way to broaden coverage; keep the authority weight low for press and social so they enrich the corpus without distorting the trend ranking.

## Session 5, 2026-06-02: Dashboard overhaul, the agentic briefing, Watch, and the scalable harness

- **Completing the multi-agent harness is mostly thin pointer files, and it is the cheapest high-leverage work in the repo.** The hub-and-spoke was half-built (AGENTS.md said Cursor and Copilot "when added"). Mirrored the civil gold standard: `.cursor/rules/project-context.mdc` (`alwaysApply: true`, `mdc:` links), `.github/copilot-instructions.md`, `opencode.md`, and `GEMINI.md`, each a tiny pointer to CLAUDE.md that names the Core Contracts and the self-improvement rule, and tells non-Claude agents to read past the Claude-only Pacing Constraints. Added `docs/FEATURE_PLAYBOOK.md` (the repeatable onboarding recipe with a fully-RAG-integrated path) and a "Sources of Truth and How to Update Them" routing table in CLAUDE.md. How to apply: when a sibling project has a harness pattern this one lacks, port it; thin pointers never drift because the substance lives in exactly one file, and a routing table is what makes "commit the lesson to the right doc" actionable rather than aspirational.

- **The CSP silently breaks third-party embeds in production while dev looks perfect.** `netlify.toml`'s CSP was `img-src 'self' data:` with no `frame-src`, so YouTube thumbnails AND the click-to-play iframe are both forbidden in production, but Vite's dev server sends no CSP, so the Watch feature looked complete in `npm run dev`. The fix is two grants on the one CSP line: `img-src https://i.ytimg.com` and `frame-src https://www.youtube-nocookie.com`. How to apply: any feature that loads a cross-origin image, iframe, font, or fetch needs the matching CSP directive, and the only honest test is a deploy preview (or replicating the header), never `npm run dev`. This is now in the FEATURE_PLAYBOOK closing checklist.

- **A build-time resolver that hits the network must degrade to a committed seed, because Netlify build IPs are datacenter IPs.** `scripts/build_creators.mjs` polls YouTube RSS, which is flaky from datacenter IPs (the same reason the ingest adapter retries). It runs as a `prebuild` step, so a hard failure would fail every deploy. Two guards: the resolver catches per creator and falls back to the corpus (`worker/.cache/items.json`, which already holds the uploads), and the `prebuild` script ends in `|| echo`, so even an unexpected non-zero exit cannot fail `npm run build`; Vite then ships the committed `creators.json`. How to apply: a build step that depends on an external feed must never be load-bearing for the build; make it best-effort with a committed fallback and exit 0.

- **Separate the presentation registry from the ingestion registry, or a UI choice perturbs the model.** Featuring a creator on Watch is a presentation decision; `youtube_channels.json` carries `authority_weight`, which scales the rotation math. A `featured: true` flag there would couple "show this on the dashboard" to "move the boards harder." Added a distinct `sources/featured.json` instead. How to apply: when a new surface wants to reuse an existing registry, check whether that registry has load-bearing side effects (weights, thresholds); if so, give the presentation concern its own file.

- **The agentic-summary pattern is portable across the trilogy: rich state, a forced-tool synthesis, sanitize, cite, and cache on a state hash.** The daily briefing (`briefing.mjs`) mirrors aigamma's narrator: build a structured state object from the window's movers, breakouts, and new items; one Sonnet synthesis via the structured-output tool (severity, headline, body, cited concepts, item refs); sanitize through `worker/lib/text.mjs`; resolve citations to concept hubs and source items; and cache the result on a hash of the state so an unchanged window costs nothing (the content-hash ethos applied to model output). The em-dash style test now scans the briefing headline and body. How to apply: for any model-written prose, never trust the prompt's style rule alone (sanitize at the boundary and unit-test the tripwire), always make the claim traceable (the Citation Contract), and gate the call on a state hash so re-runs are free and deterministic-enough.

- **One concept can rotate on more than one kind's board at once, so a unified chart must key by `kind::id`, not `id`.** Plotting all three kinds on one plane surfaced "test-time compute" as both a technique (leading) and an opinion (lagging); keying React nodes and the hover and visibility state by the bare concept id collided them. The composite `uid` fixes it while navigation still targets the shared `/technique/:slug` hub. How to apply: when merging per-kind data into one view, the row key is the (kind, id) pair, not the id; the id alone is only unique within a kind.

- **Pruning the long tail belongs in a named registry rule, and "never drop a kind to zero" is the load-bearing clause.** The unified plane keeps the top per-kind by attention then an interest gate (breakout, quadrant jump, or far enough off the neutral 100), capped, all from the `ROTATION` constant in `registry.js`. The clause that keeps it honest on a quiet kind is the fallback to that kind's single strongest entity, so a sparse board still shows one representative instead of vanishing. Verified against the real week artifacts (14 plotted from 48). How to apply: put display thresholds in the registry (one place to tune and document), and always include a floor so a filter cannot empty a category.

- **Removing a feature's last consumer orphans its component; delete it and fix the comment that named it.** Collapsing the three per-kind boards into one plane left `RotationBoard.jsx` with no importer (grep confirmed; KindView uses `RotationChart` directly), and a comment in `RotationChart` still said "RotationBoard surfaces them separately." Deleted the component and corrected the comment in the same change. How to apply: this is the Session 4 lesson again; after a refactor, grep the changed component's old collaborators for now-dead code and now-false comments.

## Session 4, 2026-06-02: Feature pruning (remove the Constellation)

- **Cut a feature that lacked a path to coherence, and take its now-dead infrastructure with it.** The Constellation (a 2D PCA map of the concept space on the Home page) was removed at Eric's call: it did not cohere and was occupying space wanted for new features. The removal spanned the component (`Constellation.jsx`), the served artifact (`constellation.json`), the worker step in `run.mjs`, the seed generator, the integrity check, and every doc reference. Its sole-purpose math (`pca2d` in `precompute.mjs`, plus its test) was dead the moment the step went, so it was deleted too, recoverable from git history if a future projection feature wants it. How to apply: when removing a feature, grep its whole footprint (component, artifact, generator, seed, tests, docs) and follow the dependency graph to anything left with no other consumer; a tested-but-unused module is exactly the incoherence the removal was meant to reduce.

How to apply: a feature that consumes a shared artifact is not the artifact's owner; confirm the producer and every other consumer before deleting upstream.

## Session 3, 2026-06-02: Deploy (GitHub, Netlify, DNS) and a CI gate

- **"Commits not getting pushed" was root-caused to there being no git remote and no upstream ever configured.** With no `origin`, `git push` had nothing to push to: it either errored or no-op'd, and the surrounding tooling reported success anyway, so finished work sat committed-only and the deployed tree silently diverged from local. The standing fix: push a new branch with `git push -u` to set the upstream, and ALWAYS verify after pushing that local HEAD equals the remote. `git status` must show `## main...origin/main` with nothing ahead (or run `git rev-parse HEAD` against `git rev-parse origin/main`). Treat a push as unverified until the local and remote SHAs match; never trust a "pushed" report on its own. The remote now exists at `https://github.com/aigamma/ai-firehose.com` (public, MIT).

- **A hermetic CI gate is cheap insurance against exactly this kind of drift.** Added `.github/workflows/ci.yml`: on push and PR to `main`, one ubuntu job runs `npm install` (no committed lockfile, so install not `ci`), `npm test`, then `npm run build`. It uses `permissions: contents: read`, a cancel-in-progress concurrency group, and no secrets. The `mcp/` smoke test is deliberately excluded: it does live network I/O and `mcp/` may be untracked on a clean clone, so it would be non-hermetic. Verified locally that both `npm test` (28 passing) and `npm run build` succeed with no env vars before wiring the workflow.

- **`docs/Commands` and `package.json` scripts must agree: an unbacked command is a doc lie.** The docs reference `npm run network`, but no such script exists, and `worker/pipeline/network.mjs` cannot back one: it only `export`s pure functions (no `main()`, no CLI entry), and `computeSpectrums` needs a live Voyage `embedFn`. All the orchestration (load the store, embed, write every dependent artifact) lives in `run.mjs`, so the network rebuild is a stage of `npm run ingest`, not a standalone command. Did NOT add a fake `network` script; the discrepancy is the doc's to fix. How to apply: before documenting a command, confirm a script (or a runnable entrypoint) actually backs it; grep `package.json` scripts for any `npm run X` a doc mentions.

- **The MCP package must not depend on this repo.** `mcp/package.json` listed `"ai-firehose": "file:.."`, which ties the published npm package back to the monorepo, breaking both `npx ai-firehose-mcp` for end users and `npm publish`, and defeating the portable, no-repo design (the server wraps the public API and needs only `@modelcontextprotocol/sdk`). Removed it. How to apply: a package meant to be published standalone must declare only real, published dependencies; a `file:` dependency is a local-dev convenience that does not survive publication.

## Session 2, 2026-06-02: Overnight hardening (split the heavy JSON payloads)

- **Precomputed artifacts are a served payload, not a scratch dump: ship only what a page renders.** The glossary `index.json` had grown to 1.08 MB because it carried the full hub (definition, neighbors with scores, axis_positions, top_items) for all 427 concepts, and both the glossary list and every `/technique/:slug` hub downloaded the whole thing to use a slice of it. `spectrums.json` was 591 KB because each axis embedded its 1024-dim `axis_vector` (needed only by a not-yet-built server projection) plus a raw `position` next to the `position_normalized` the UI actually reads. `neighbors.json` (287 KB) was served but fetched by nothing (already denormalized into the hubs). Why it matters: this is a static site, so every committed artifact is downloaded by real visitors, and the heaviest sat on the two most data-rich pages. How to apply: split per-entity hubs into `glossary/c/<id>.json` fetched on demand, keep a slim `index.json` (`id, label, kind, attention, aliases, def_snippet`) for list and search, strip any field no consumer reads, and park server-only data (axis vectors) outside `public/`. Result: glossary list 1.08 MB to 88 KB, each hub about 4 KB, spectrums 591 to 398 KB, neighbors removed. The fix touched both the generator (`run.mjs`) and the already-committed artifacts (`worker/pipeline/slim_artifacts.mjs`, an idempotent one-time migration), because fixing only the generator would leave the live tree heavy until the next ingest.

- **Audit who actually fetches an artifact before adding fields to it.** Grep each artifact name across `src/` and `netlify/`. A field only the precompute step or a future feature needs does not belong in the browser payload. Caveat learned here: the dedicated ripgrep-based search tool respects `.gitignore`, so it silently skips the built `dist/` bundle and even returned empty for strings that exist in tracked source during this session; a raw recursive `grep` is the authoritative cross-check before deleting or reshaping a served file.

- **The MAX backstop and per-source limits are working: a real run produced 258 items across 6 of 7 sources.** YouTube 104, Hacker News 32, arXiv 30, blog 34, GitHub 25, Hugging Face 33; Reddit still 0 from this datacenter IP (expected, works from Fly or residential). 427 concepts after AI-grown resolution. How to apply: this is a healthy reference distribution to sanity-check future runs against; a sudden collapse in one source's count is an adapter regression, not a quiet day.

- **Do not prematurely declare "complete" and drop into standby: a fresh-eyes audit of an already-"finished" site found eight commits of genuine work.** Re-reading the actual artifacts instead of trusting a prior "done" claim surfaced a 1 MB glossary payload, two spec'd-but-missing hub fields (momentum, first_seen), a silent concept-loss bug, and a stale Methodology paragraph. Why it matters: the loop's value is real improvement, and idle "standby" with tight pulses is exactly the theater the global harness rules forbid. How to apply: when tempted to stand by, instead audit one concrete dimension end to end (payload sizes, referential integrity, spec vs implementation, a house rule) and measure it. Either you find real work or you bank a verifiable green check, both better than a pulse. When the safe, locally-verifiable backlog is genuinely exhausted, keep the loop alive on a long heartbeat rather than a 250s pulse, and say so plainly.

- **A prompt instruction is not enforcement: enforce house rules in code.** Every model prompt says "no em dashes", yet models emit them anyway, and the displayed prose (classifier summaries, glossary definitions) had no guard. Added `stripEmDashes` (`worker/lib/text.mjs`) applied in `classify.mjs` and `define.mjs` so model output is sanitized before it enters the corpus. Why it matters: the committed definitions happened to be clean, but that was luck, and summaries live only in Pinecone where the artifact test cannot see them. How to apply: when a house rule governs model output, add a deterministic post-processor at the point the output enters the system, and unit-test the processor; treat the prompt instruction as a hint, not a guarantee. En dashes (numeric ranges) and verbatim source titles are deliberately left alone.

- **The classifier can emit `concepts` as a comma-joined string, not an array.** One stored item carried a string; `canonicalizeConcepts` iterates with for..of, so it walked the string character by character and dropped every char as junk, silently losing that item's concepts (no crash, because `remap` returns an array). How to apply: concepts are now coerced at the single chokepoint (`toConceptList` in `concepts.mjs`). If a future change reads raw `item.concepts` elsewhere, coerce there too, and watch for the same shape drift in `entities`.

- **An accumulating store is durable state, so it must be committed, not gitignored.** `worker/.cache/items.json` (the rolling-quarter corpus, the substrate for every rebuild) was gitignored as if it were a cache, but the worker clones the repo fresh each run with no persisted volume. So neither the Fly worker nor a CI runner would accumulate the quarter: each run starts empty, the corpus collapses to the current feed snapshot, and the Pinecone reconcile then deletes everything else. OPERATIONS.md's recovery recipe even assumed the raw items were staged. Fix: commit `items.json` (retention bounds it to ~0.5 MB) and add it plus `feed.xml` to the publish commit-back; the genuinely regenerable caches (classify, definitions, axis_vectors) stay ignored. How to apply: separate durable state from cache by lifecycle, not by directory. If losing a file silently degrades the system across runs, it is state and must persist.

- **An accumulating store keyed by content hash silently duplicates re-edited source items.** A YouTube video re-titled between runs (creators A/B test titles constantly) gets a new content hash, hence a new deterministic id, and `store[id] = it` added it alongside its stale prior version; retention only prunes by `published_at`, which a re-title does not change. The store had two entries for one video ("Anthropic is about to IPO" and "GPT-5.6 about to DROP", same URL). Over a quarter of daily runs this compounds, double-counting a popular video's concepts in the rotation and cluttering the RSS feed. Fix: `collapseStore` (`worker/pipeline/store.mjs`) keeps one entry per `(source, source_id)`, preferring this run's freshly classified version, else the most recently published. How to apply: any store keyed by a content hash needs a collapse step on the stable source key; the within-run dedup (which keys on `source:source_id`) does not catch cross-run re-edits.

- **Batch embedding and rerank APIs tag each result with its input `index`; reorder by it, never trust array order.** `voyage.mjs embed()` pushed embeddings in response order, which would silently misalign a whole batch with its inputs if the API ever reordered, corrupting every similarity, cluster, neighbor, and spectrum, the most catastrophic-if-wrong path in the system. Fixed with `orderEmbeddings` (place each at `d.index`). How to apply: the same holds for any OpenAI-style batch endpoint; Voyage rerank already maps via `r.index` in `retrieve.mjs`, so that path was already safe.

- **Audit the docs against the implementation, not only the code against itself.** CLAUDE.md's core "Classification Without Guessing" and "AI-Grown Taxonomy" contracts described a `_proposed.json` human-approval staging gate, an OpenAI dual-check, and registry-driven thresholds (0.86/0.78), none of which the resolver actually did: `concepts.mjs` hardcoded 0.9/0.8, never read the registry, and auto-entered every new concept with no staging. The keystone protocol treats docs that lie as a process failure equal to code that lies, so this is not cosmetic. Resolution this session: wired the registry `TAXONOMY` thresholds into the resolver so the single source of truth is real, then rewrote CLAUDE.md, the registry comment, and `docs/INGESTION.md` to describe the implemented algorithm and to mark the approval gate and dual-check as intended-but-not-implemented. How to apply: when a contract names a file or constant (`_proposed.json`, `mergeThreshold`), grep for it; an unreferenced name is either dead config or a lie. Prefer implement-to-match when it is safe; choose doc-to-match when the code is sound and re-running to validate a behavior change is not (here: no keys, and a threshold change re-clusters the whole taxonomy).

## Session 1, 2026-06-01: Founding (planning and scaffold)

The decisions and principles that shaped the project, captured so a fresh agent inherits them without re-deriving them.

- **The core is robotlogic.org's non-chat embedding machinery, not a chatbot.** The most advanced RAG work in the portfolio (constellation, centroids, clusters, concept spectrums, influence graph, neighbors) is all non-conversational. Why: that machinery organizes and visualizes a corpus, which is exactly what this site needs. How to apply: do not add a chatbot. The only live model surface is semantic search via `netlify/functions/retrieve.mjs`.

- **Retention is a feature: the corpus self-expires at one quarter.** Why: in the fastest-growing industry, stale signal is noise, and a bounded corpus keeps Pinecone and Voyage costs flat forever. How to apply: the prune stage deletes by `published_at` past `RETENTION_DAYS` (registry). The four horizons are nested windows inside that single quarter; Quarter is the hard outer bound.

- **The taxonomy is AI-grown and fitted loosely, never a fixed tag list.** Why: a fast field coins new concepts constantly, and exact-string tagging both misses them and fragments near-duplicates. How to apply: the model discovers and names new concepts, then every candidate is reconciled to the existing taxonomy by voyage-3 cosine (`TAXONOMY` thresholds in registry). Bind at or above 0.86, review in the ambiguous band, create-new below 0.78 into `_proposed.json` for in-repo approval.

- **Three per-kind rotation boards, and the rotating entity differs per kind.** Techniques rotate by concept, Tools by named product or repo, Opinions by discourse theme. Why: Eric chose three boards plus one unified constellation. How to apply: one shared attention-and-rotation computation, three entity sources. (Superseded Session 4: the unified constellation was removed; the three rotation boards remain.)

- **YouTube is the primary source, coded from scratch.** Channel RSS for new-upload detection, yt-dlp for metadata and captions, Whisper for caption-less videos (the exact civil transcript pattern). Why: high-signal teachers are a leading indicator, and from-scratch is cheaper and more interesting than a paid scraper. How to apply: Apify is documented only as a paid fallback if RSS or yt-dlp is blocked. Favorite teachers get high `source_authority_weight`.

- **Public, no auth; curation happens in-repo.** Why: simplest, matches the sibling sites, and the human-author stop (approving new glossary concepts) fits a file or PR review. How to apply: proposed concepts land in `public/data/glossary/_proposed.json`; Eric approves them into the canonical taxonomy.

- **Classification is deterministic-or-flagged, never guessed.** Why: a strict schema plus an optional OpenAI dual-check keeps hallucinated tags out. How to apply: disagreement routes an item to staging rather than publishing.

- **The AGENTS.md hub-and-spoke is what makes the ensemble safe.** Why: Eric runs Codex, Cursor Composer, and Antigravity Gemini alongside Claude. Each reads `AGENTS.md` then `CLAUDE.md` and builds against the deterministic, idempotent contracts. How to apply: keep `AGENTS.md` a pure delegator; per-model driving prompts live in `D:\prompts`.

- **Pacing: parallel subagents for independent build-out, not `/loop`.** Why: an N-item backlog finishes in the time of the slowest single subagent. How to apply: fan out independent adapters or component ports as parallel `Agent` calls; reserve the main thread for coherent, cross-cutting work like the steering docs.

- **The plan lives in the planning system; its decisions are mirrored into `CLAUDE.md`.** Why: durable conventions belong in the auto-loaded doc, not only in a plan file. How to apply: when a decision changes, update `CLAUDE.md` and note it here.

- **Lean SVG over Plotly for the rotation and constellation charts.** Why: the rotation plane is a four-quadrant scatter and the constellation is a 2D scatter; neither needs a multi-megabyte charting engine, and the bundle stays near 58 kB gzip. How to apply: keep visualizations as small SVG components (`RotationChart.jsx`, `Constellation.jsx`, `Sparkline.jsx`) unless a feature genuinely needs a charting library. This refines the plan, which had named Plotly. (Superseded Session 4: `Constellation.jsx` was removed; `RotationChart.jsx` and `Sparkline.jsx` remain the live SVG charts.)

- **Built the dashboard against a clearly-labeled synthetic seed before the live pipeline.** `worker/seed/seed.mjs` emits artifacts with `synthetic: true`, and the UI shows a "demo data" ribbon. Why: makes the outlier-hunt concept tangible and surfaces design feedback early without blocking on real data, keys, or the channel list. How to apply: the schemas the seed emits (in `docs/RAG.md`) are the contract the real worker must honor; swap to real artifacts in Phase 1+ and the ribbon disappears on its own.

- **Never call the rotation visualization "RRG" or a "Relative Rotation Graph".** Those name a trademark (de Kempenaer / StockCharts). Why: Eric deliberately cites older prior-art math and flagged this directly. How to apply: use "rotation plane", "rotation ratio", and "rotation momentum", and cite Mansfield Relative Performance (Roy Mansfield, 1979), exactly as aigamma.com/rotations does. This is also in CLAUDE.md Writing Rules.

- **The full live path is verified, so do not treat external inputs as a blanket blocker.** A smoke run (`worker/pipeline/smoke.mjs`) proved fetch -> Claude classify -> Voyage embed -> Pinecone upsert -> query on real videos, and Claude's kind and concept extraction is good out of the box. Why: this was logged after a session wrongly stopped, claiming the channel list blocked all of Phases 1 to 3 when it only gated the YouTube adapter. How to apply: build and verify the autonomous surface (substrate, rotation, precompute, non-YouTube adapters) without waiting; only the YouTube channel curation needs Eric.

- **YouTube channel ids: resolve from the canonical/RSS-alternate link, and the feed endpoint is flaky from datacenter IPs.** The first `channelId` in the page blob is sometimes a different channel. The RSS feed returns intermittent 404 and 5xx for valid channels, so retry with backoff (the adapter does). How to apply: trust a 200 with content; do not delete a channel on a single 404.

- **The ai-firehose Pinecone index was created with the civil project key, so it lives alongside civil-rights and worldthought.** It is isolated by index name (host ai-firehose-...pinecone.io). Why: Pinecone projects cannot be created via an API key. How to apply: optionally migrate to a dedicated ai-firehose project later; for now never write to the sibling indexes (pinecone.mjs resolves the host from the index name, not a sibling PINECONE_HOST).

- **Sparse, bursty attention breaks raw rotation math; smooth it into a decayed level first.** On the first real run a concept seen in one video on one day produced ratio 750 and momentum 237, because Mansfield normalization assumes a continuously-sampled, price-like series. Fix: `decayedLevel()` (attention.mjs) turns mentions into a smooth, fading attention level (half-life about 10 days) that the rotation engine consumes; the displayed attention stays the raw weighted mentions in the window; ratio and momentum are clamped to [55, 145] as a safety. How to apply: always feed levels, not raw counts, to the rotation engine. After the fix, test-time compute reads ratio 62.8, momentum 79.6 with a smooth sparkline.

- **Concept near-duplicates still need the embedding fuzzy-merge (next priority).** The first real run produced llm, large-language-models, and generative-ai as separate concepts. This is exactly what the AI-grown taxonomy resolution (docs/INGESTION.md, the TAXONOMY thresholds) is meant to collapse by voyage-3 cosine. It is the top Phase 2 task; until it lands the boards and constellation carry some duplication.

- **Concept resolution shipped (concepts.mjs) with a lexical guard.** Pure cosine over-merged short "AI X" labels (self-improving AI fused into generative AI) because they embed very close. The rule: merge on very high cosine (>= 0.90, near-identical such as case variants) OR moderate cosine (>= 0.80) plus a lexical signal (a shared significant token, or one label is an acronym of the other, so LLM binds to large language models). This kept the real merges (LLMs, AI agents) and dropped the loose ones. Inspect public/data/glossary/_concepts.json to retune.

- **Artifacts build from an accumulating, retention-pruned store, not a single fetch.** YouTube feed flakiness means each run fetches a different subset (saw 30 to 60 items), so building from one run made the dashboard thrash. Fix: worker/.cache/items.json accumulates classified items keyed by id, pruned by published_at past RETENTION_DAYS; attention, constellation, and digests build from the whole store. A classify cache (worker/.cache/classify.json) keeps re-runs free. In the loop, running the pipeline repeatedly unions the flaky fetches; the corpus grew 45 to 60 over three runs and will approach full channel coverage overnight. Follow-up: Pinecone still holds aged-out vectors (the local store prunes, the index does not yet); add an index reconcile.

- **Network precompute shipped (network.mjs).** Per-concept cosine neighbors, deterministic k-means clusters (cosine geometry, fixed init, no Math.random), concept-axis spectrums (7 AI-discourse axes with the axis_vector persisted for live projection; anchors in prompts/axes.mjs), and a co-occurrence influence graph. Real clusters emerged, for example a business-of-AI cluster (PIV Loop, vendor lock-in, founder leverage) reflecting the Saraev and Ottley channels. The Explore UI to surface these is the next step.

Minor under-merging remains (singular vs plural, LLMs vs large language model); acceptable for now, retune the lexical guard later.

- **The Explore and Glossary UI reads only static artifacts (no function needed).** The worker emits glossary/index.json with each concept as an integration hub (aliases, kind, attention, neighbors, axis positions, recent items, the worldthought person-page-as-hub pattern). Hacker News is gated by an AI keyword regex so Claude is not paid to classify unrelated stories; arXiv retries with backoff because it returns 429 to aggressive callers. Dedupe is keyed by source plus source_id (ids can collide across sources).
