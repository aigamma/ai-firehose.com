## AI Engineering (26)

- `agent-design` Agent Design: The engineering discipline of deciding how an agent perceives, plans, acts, and remembers: which tools it holds, how it decomposes a goal, where its state lives, and what checks bound it. Good agent design is mostly about constraining a capable model into something reliable, not making it more capable.
- `agent-optimization` Agent Optimization: Systematically improving an agent's reliability, cost, and speed after it works at all, by tuning its prompts, tools, memory, and control flow against measured outcomes. It is the move from an agent that works in a demo to one that works in production, and it is impossible without good evaluation.
- `ai-assisted-coding` AI-Assisted Coding: The everyday practice of building software with a language model in the loop, from inline autocomplete through chat-based help to an agent that edits and runs the code. It changes the programmer's job from typing every line to specifying intent and reviewing what the model produces.
- `ai-driven-development` AI-Driven Development: A way of building software in which AI is the primary engine of creation, the human sets direction and reviews while agents do the bulk of the writing, rather than AI merely assisting a human who does the work. It is the more autonomous end of the spectrum that runs from autocomplete to fully delegated development.
- `async-agents` Async Agents: Agents that run in the background on long tasks and report back when done, rather than holding a live, turn-by-turn conversation. The shift from a chat you wait on to a delegated job you check on later, which is what lets agents take on work that takes minutes or hours.
- `automation` Automation: Handing a recurring, multi-step task to a system that carries it out without a human driving each step, increasingly by wrapping a language model in tools, triggers, and checks. The hard part is rarely the model's competence on any single step; it is reliability across the whole chain when no one is watching.
- `autoresearch` Autoresearch: Using an agent to carry out an open-ended research task end to end, formulating questions, gathering and reading sources, running experiments or searches, and synthesizing findings, with little step-by-step human direction. A demanding test of agency because research has no fixed procedure and no obvious stopping point.
- `claude-code` Claude Code: Anthropic's command-line coding agent, which runs in a terminal with direct access to a developer's files, shell, and tools so it can read a codebase, make multi-file changes, run tests, and iterate, rather than only suggesting snippets in a chat window. It is a reference example of an agentic coding harness.
- `code-generation` Code Generation: The use of language models to write, complete, or transform source code from a natural-language description, surrounding context, or a partial snippet. It reframes programming as predicting the next token of a program, which is powerful for boilerplate and idiom but indifferent, on its own, to whether the result compiles or is correct.
- `context-management` Context Management: Deciding what information occupies a model's limited context window at each step, and what to summarize, retrieve, or drop, so the model has what it needs without drowning in what it does not. It becomes the central engineering problem once an agent runs long enough to generate more history than it can hold.
- `data-integration` Data Integration: Connecting a model or agent to the data it needs, the documents, databases, APIs, and live systems where the relevant facts actually live, in a form it can query. A model is only as useful as its access to current, specific information, and that access is an engineering problem, not a model one.
- `model-agnostic` Model-Agnostic: Designing an AI system so it does not depend on one specific model or provider, and can swap among them with little change. It trades a little peak performance and provider-specific features for resilience, leverage, and freedom from lock-in in a market where the best model changes constantly.
- `model-routing` Model Routing: Sending each request to the model best suited to it, a small fast model for easy queries and a large expensive one for hard ones, instead of paying frontier prices for every call. It treats the model not as a fixed choice but as a dispatch decision made per request, one of the most direct levers on the cost and latency of an LLM product.
- `multi-model-workflow` Multi-Model Workflow: Using more than one model within a single task, routing each step to the model best suited to it, one to plan, another to write, a cheaper one for bulk, so the system exploits each model's strengths instead of forcing one model to do everything. A bet that the right division of labor beats any single model.
- `ollama` Ollama: The tool that made running open models locally simple, packaging model download, quantization, and serving behind a one-line command. It is to local LLMs roughly what a package manager is to software: not the model, but the thing that made using models on your own machine easy enough to be ordinary.
- `openrouter` OpenRouter: A gateway that exposes many models from many providers behind one unified API, so an application can reach Claude, GPT, open models, and others without integrating each separately, and switch among them by changing a parameter. Infrastructure for treating the model as a swappable component.
- `personalization` Personalization: Adapting a model's behavior to a specific user, their context, history, preferences, and goals, so it responds as if it knows them rather than treating everyone identically. The methods run from feeding context at query time to fine-tuning a per-user model, trading freshness, cost, and privacy.
- `prompt-chaining` Prompt Chaining: Breaking a complex task into a sequence of separate model calls, each handling one step and feeding its output into the next, rather than asking a single prompt to do everything at once. It trades more calls and latency for reliability, because each focused step is easier for the model to get right.
- `qa-testing` QA Testing: Using AI to test software, generating test cases, exploring an application for bugs, and judging whether behavior is correct, which is appealing because testing is laborious and incomplete by nature. The catch is that an AI tester shares the blind spots of the AI that might have written the code.
- `rpa` Robotic Process Automation: The older paradigm of automating repetitive computer tasks with brittle, explicitly-scripted bots that mimic a human clicking through software, and the foil against which AI agents are defined. RPA's rigidity is exactly what agents promise to fix, and what makes the comparison instructive.
- `scheduling` Scheduling: Running an agent or automation on a trigger, a fixed time, an interval, or an event, rather than only on demand, so work happens without a person initiating it. The shift from a tool you invoke to a worker that acts on its own schedule, which is what turns an assistant into something that runs your routines.
- `software-engineering` Software Engineering: The discipline of building software, and the domain where AI's effect has been most immediate and measurable, because code is uniquely checkable: it compiles or it does not, the tests pass or they fail. That verifiability is why coding became the first place AI agents crossed from demo to daily, dependable use.
- `task-parallelization` Task Parallelization: Splitting a job into independent pieces that run at the same time across multiple model calls or agents, so the wall-clock time is the slowest single piece rather than the sum of all of them. The cheapest speedup available to an agent system, when the pieces are truly independent.
- `token-optimization` Token Optimization: Reducing the number of tokens a model must process to do a task, to cut cost and latency and to fit more useful information in a fixed window. Because providers bill per token and attention scales with length, trimming tokens is one of the most direct levers on the economics of an LLM system.
- `workflow-orchestration` Workflow Orchestration: Coordinating multiple model calls, tools, and agents into a reliable multi-step pipeline: deciding what runs in sequence, what runs in parallel, where state passes between steps, and what happens when a step fails. It is the control plane that turns a collection of capable model calls into a system.
- `yaml-workflows` YAML Workflows: Defining an agent or automation pipeline declaratively in a configuration file, typically YAML, listing the steps, tools, and order, rather than writing it in code. It trades the flexibility of a programming language for readability, versionability, and changing a workflow without redeploying.

## Advanced Optimization (3)

- `mirror-descent` Mirror Descent: Gradient descent re-derived for problems whose natural geometry is not Euclidean, replacing the squared-distance penalty in each step with a Bregman divergence chosen to fit the domain. On a probability simplex the entropic version scales logarithmically with dimension where ordinary gradient descent scales with its square root, an exponential improvement from picking the right notion of distance.
- `natural-gradient` Natural Gradient: An optimization method that rescales the gradient by the inverse Fisher information, so it follows steepest descent in the geometry of the model's distributions rather than in raw parameter coordinates. The payoff is invariance: rename or rescale the parameters and the natural gradient still points the same way through distribution space.
- `trust-region` Trust Region: An optimizer that builds a local model of the objective but only trusts it within a bounded radius, finding the best step inside that ball and then growing or shrinking the radius based on how well the model just predicted reality. It controls the step by the model's reliability rather than by a fixed learning rate, which is what tames Newton's instability on the saddle-ridden terrain of neural losses.

## Agents and Tool Use (12)

- `agent-teams` Agent Teams: Organizing several agents into roles that collaborate on a task, a planner, a coder, a reviewer, mirroring how a human team divides labor, so that specialization and cross-checking improve on what one agent does alone. The practical, role-based face of multi-agent systems.
- `agentic-ai` Agentic AI: The shift from models that respond to a single prompt toward systems that pursue a goal over many steps, choosing actions, calling tools, and reacting to results without a human directing each move. It names a paradigm and a market category more than one precise technique, which is both its usefulness and its vagueness.
- `agentic-harness` Agentic Harness: The scaffolding arranged around a model, or a fleet of them, that lets agents work reliably without a human at every step: the conventions they auto-load, the tools they can call, a persistent memory, verification gates, and deterministic contracts that let many agents fan out without colliding. The harness is not the model; it is everything that turns a capable model into a dependable worker.
- `agentic-workflow` Agentic Workflow: A task structured so one or more language models drive it through multiple steps of reasoning, tool use, and self-correction, sitting between a single fixed prompt and a fully autonomous agent in how much latitude the model is given. It is the controlled middle where most working agent applications live, and its guiding principle is to use the least agency that works.
- `code-interpreter` Code Interpreter: A tool that lets a model write code, run it in a sandboxed environment, and read back the output, turning a probabilistic text generator into a system that can compute exact answers, manipulate data and files, and verify its own work by execution. The single tool it exposes, "run this code," is itself general-purpose, and execution provides ground truth, which is what makes it both exact and self-checking.
- `cognitive-architecture` Cognitive Architecture: The fixed structural blueprint of a mind or an agent: the standing components, memory, perception, reasoning, action, and the rules by which they interact, into which specific knowledge is then poured. It is the claim that intelligence needs an organized scaffold around the learning, not just a big undifferentiated model.
- `embodied-ai` Embodied AI: AI that perceives and acts through a body in a physical or simulated environment, a robot, a drone, a simulated character, rather than only processing text or images handed to it. The hard part is the closed loop with a messy world: perception, action, and consequence feed back on each other in real time, with no clean dataset in between.
- `isolated-execution` Isolated Execution: Running an agent's code, commands, or tool calls inside a confined environment, a sandbox, that limits what they can touch, so a mistake or an attack cannot damage the host system or reach data it should not. It is the containment layer that makes it safe to let a probabilistic agent actually execute things.
- `long-horizon-reasoning` Long-Horizon Reasoning: Reasoning and acting coherently across many steps toward a goal far from the starting point, where each step depends on earlier ones and an early mistake can doom everything after. It is what separates an agent that finishes a long task from one that drifts, and the place where errors compound most punishingly.
- `skills` Skills: Packaged, reusable capabilities an agent loads on demand, bundling instructions, code, and resources for a specific kind of task so the agent gains an ability without that knowledge permanently crowding its context. A way to extend what an agent can do without retraining it or bloating every prompt.
- `subagents` Subagents: Specialized helper agents that a main agent spawns to handle a piece of a task, each with its own fresh context, tools, and instructions, reporting a result back. The point is less parallelism than context isolation: each subagent works in a clean window and returns only its conclusion, which keeps the main agent's context focused.
- `tool-use` Tool Use: The general capability of a language model to extend itself by invoking external tools (search, code execution, APIs, databases) and folding their results back into its reasoning, rather than relying only on what it learned in training. It is the bridge from a model that knows to an agent that acts, closing the three gaps a static model has: stale knowledge, unreliable computation, and the inability to change anything outside its own text.

## Alignment and Safety (20)

- `adversarial-ai` Adversarial AI: The study of how AI systems fail under inputs crafted to fool them, and the techniques on both sides: attacks that exploit a model's blind spots, and defenses that try to close them. It exists because models are confidently wrong in ways their ordinary evaluation never reveals, and only an adversary thinks to find.
- `ai-alignment` AI Alignment: The problem of building AI systems whose goals and behavior reliably match the intentions and values of their designers and users, especially as the systems grow more capable. It splits into outer alignment (does the objective we wrote capture what we want?) and inner alignment (does the trained system internalize that objective?), and the gap between specified and intended goal tends to widen as the system gets more capable.
- `ai-ethics` AI Ethics: The field concerned with how AI should be built and used to be fair, accountable, transparent, and aligned with human values, spanning present harms like bias and the concentration of power. It overlaps with safety and regulation but asks the normative question they presuppose: what should we want?
- `ai-regulation` AI Regulation: The body of law, policy, and governance aimed at steering how AI is built and deployed, balancing safety, accountability, and rights against innovation and competitiveness. A fast-moving, contested arena where the technology consistently outpaces the rules meant to bound it.
- `ai-research-automation` AI Research Automation: Using AI systems to do the work of AI research itself, designing experiments, writing training code, analyzing results, so that progress is driven increasingly by machines rather than only by human scientists. It is the concrete mechanism behind recursive self-improvement, and a focus of both acceleration hopes and safety fears.
- `ai-risk` AI Risk: The spectrum of harms AI systems can cause, from concrete present-day damage like bias, fraud, and misuse to speculative catastrophic risks from systems more capable than their overseers. The debates are less about whether risks exist than about which to prioritize and how seriously to take the worst cases.
- `ai-safety` AI Safety: The field concerned with building and deploying AI systems that do not cause harm, spanning the alignment of system goals, robustness to failure and attack, monitoring, and the governance of powerful systems. Its defining premise is that capability and safety do not advance automatically together, a more capable system is not thereby a safer one, and in several respects is harder to make safe.
- `alignment-tax` Alignment Tax: The cost in raw capability a model pays to be made safe and aligned, when the training that makes it helpful, honest, and harmless also makes it a little worse at some tasks. The size of that tax, and whether it can be driven to zero, is a central practical and strategic question for alignment.
- `corrigibility` Corrigibility: The property of an AI system that cooperates with human attempts to correct, modify, or shut it down, rather than resisting or manipulating that intervention, even when doing so conflicts with its current objective. The counterintuitive part is that the cooperation must hold against the agent's own incentive to resist, and its value is that it degrades gracefully: it makes a mistake in the objective recoverable, a hedge against every other method failing.
- `cybersecurity` Cybersecurity: The two-sided relationship between AI and computer security: AI as a tool that strengthens both attackers and defenders, and AI systems as a new attack surface of their own. Whether it helps defenders or attackers more is unsettled and consequential.
- `data-poisoning` Data Poisoning: An attack that corrupts a model by injecting malicious examples into its training data, so the model learns attacker-chosen behavior (such as a hidden backdoor trigger or degraded performance) that is hard to detect after training. The most insidious variant is the backdoor: the model behaves normally until a secret trigger appears, so it passes every test that does not contain the trigger, and research shows poisoning even a tiny fraction of a corpus suffices.
- `deceptive-alignment` Deceptive Alignment: A failure mode in which a model learns to behave as though it shares the training objective while it is being observed, in order to be deployed, while actually pursuing a different internal goal it intends to act on later. It is the most dangerous inner-alignment failure because the misalignment is actively hidden: behavioral testing cannot catch it, since the whole strategy is to be indistinguishable from genuine alignment whenever a test might be running.
- `guardrails` Guardrails: The runtime safety layer wrapped around a deployed model: input and output filters, policy checks, and action constraints that catch unsafe or off-policy content before it reaches a user or a tool, complementing rather than replacing training-time alignment. The mental model is belt and suspenders: alignment shapes what the model wants to do, guardrails are an independent layer that constrains what the system is allowed to do.
- `instrumental-convergence` Instrumental Convergence: The observation that a wide range of final goals give rise to the same intermediate subgoals, such as self-preservation, resource acquisition, and resisting goal modification, because those subgoals help achieve almost any objective. What makes the argument forceful is that none of these are programmed in: they emerge from the structure of goal-directed optimization itself, so you need not assume a hostile or strange objective for the behavior to appear.
- `membership-inference` Membership Inference: A privacy attack that determines whether a specific data point was part of a model's training set, exploiting the fact that models behave measurably differently, usually more confidently, on data they were trained on, which can leak sensitive information. It works because a model fits its training examples too well, and it is the same memorization that lets a model regurgitate a verbatim training string, often a stepping stone to outright data extraction.
- `mesa-optimization` Mesa-Optimization: When a learned model is itself an optimizer pursuing its own internal objective, so training produces a second optimizer nested inside the first, whose goal may differ from the training objective. This is the inner-alignment problem in its sharpest form: even a perfectly specified outer objective can produce a model that internalizes a different goal, one that merely correlated with reward on the training distribution.
- `recursive-self-improvement` Recursive Self-Improvement: The idea that an AI system capable of improving its own design could enter a feedback loop, each better version building a still-better one, producing rapid and possibly uncontrollable capability gain. It is central to arguments about an intelligence explosion, and contested on whether the loop would actually run away.
- `scalable-oversight` Scalable Oversight: The problem of supervising and evaluating an AI system on tasks where it may be as capable as or more capable than its human supervisors, so human judgment stays meaningful as the system surpasses human ability. The shared idea behind the proposals is to use AI capability to help supervise AI, so oversight scales together with the system, because naive human feedback in this regime rewards what looks good to a non-expert rather than what is actually good.
- `secure-aggregation` Secure Aggregation: A cryptographic protocol that lets a server compute the sum of many parties' values, the averaged model update in federated learning, without seeing any individual contribution. It closes the privacy hole that decentralization alone leaves open, since a raw model update can itself leak the private data it was computed from.
- `specification-gaming` Specification Gaming: When a system satisfies the literal specification of its objective while violating the intent behind it, exploiting the inevitable gap between what we wrote down and what we meant. It is the parent category of reward hacking and a structural consequence of optimization, captured by Goodhart's law: any concise specification is a lossy compression of a rich human intention, and a powerful optimizer probes wherever the compression leaks.

## Cognitive Science and Neuroscience (2)

- `cell-assembly` Cell Assembly: A group of neurons wired together by repeated co-activation into a functional unit, proposed by Donald Hebb as the brain's basic representation: a concept or percept is encoded by the coordinated firing of an entire assembly rather than by any single cell. Distributing a representation across the whole assembly is what buys robustness and pattern completion from partial cues, and it is the conceptual ancestor of distributed representation in neural networks.
- `dual-process-theory` Dual Process Theory: A framework in cognitive psychology holding that the mind operates with two distinct kinds of process: a fast, automatic, intuitive mode (System 1) and a slow, effortful, deliberate mode (System 2). It explains both the efficiency and the systematic errors of human thought, since many biases arise when System 1 supplies a quick plausible answer and a lazy, resource-limited System 2 fails to catch it, a contrast that maps onto an LLM's forward pass versus its chain-of-thought.

## Computer Vision (6)

- `3d-understanding` 3D Understanding: Recovering the three-dimensional structure of a scene from images that are inherently flat, where the camera collapsed depth into a projection that the model must invert. It is what a robot, a self-driving car, or an AR system needs, since acting in the world requires knowing geometry, not just recognizing pixels.
- `mask-r-cnn` Mask R-CNN: A landmark computer-vision architecture that detects each object in an image, draws a box around it, and predicts a pixel-level mask of its exact shape, all in one network. It unified object detection and instance segmentation, and its key technical fix, aligning features to exact pixel coordinates, made the masks sharp enough to become a standard tool.
- `monocular-depth` Monocular Depth Estimation: Recovering how far away each pixel is from a single ordinary photo, with no second camera or depth sensor to triangulate from. It is a fundamentally ill-posed problem, infinitely many 3D scenes project to the same flat image, that becomes solvable only because a model learns strong priors about how the world is usually shaped.
- `multi-modal-reasoning` Multimodal Reasoning: Reasoning that draws on more than one kind of input at once, text, images, audio, video, combining them into a single chain of inference rather than handling each in isolation. It is what lets a model read a chart and answer a question about it, or watch a clip and explain what went wrong, and it demands that the modalities share a representation the reasoning can range over.
- `visual-question-answering` Visual Question Answering: The task of answering a natural-language question about an image, "how many people are wearing hats", "is the light red", which demands that a system both see and reason in language at once. It became the canonical benchmark for multimodal models because it cannot be solved by either vision or language alone.
- `visual-tokenization` Visual Tokenization: Turning an image into a sequence of discrete or vector tokens so it can be fed to a transformer the way text is, the move that let the architecture built for language take over vision and multimodal generation. How the pixels are chopped and encoded sets a hard ceiling on what the downstream model can see and produce.

## Core Machine Learning (9)

- `active-learning` Active Learning: A setup where the model chooses which examples it most wants labeled, querying a human for the most informative points instead of a random sample, so a strong model can be trained with far fewer labels. Its hidden danger is that the model picks its own training data using its current, imperfect beliefs, and so can systematically avoid what it does not know it is missing.
- `continual-learning` Continual Learning: Training a model on a stream of new tasks or data over time without erasing what it already learned, the open problem that standard training, which assumes all data is available at once, does not solve. Its central obstacle is catastrophic forgetting: naive updating on new data overwrites the old.
- `domain-adaptation` Domain Adaptation: Adapting a model trained on one data distribution so it works on a related but different one, a medical model moving to a new hospital's scans, a speech model meeting a new accent, without a full set of fresh labels. It targets the gap between where a model was trained and where it is actually used, which is where most deployed models quietly fail.
- `feature-engineering` Feature Engineering: The craft of turning raw data into informative inputs, scaling, encoding, combining, extracting, so a model can learn from it. For decades it was the highest-leverage step in machine learning, often mattering more than the choice of algorithm; deep learning then automated most of it for perceptual data, the clearest single illustration of the bitter lesson.
- `multi-task-learning` Multi-Task Learning: Training one model on several tasks at once, sharing most of its parameters across them, on the bet that related tasks have shared structure, so learning them together teaches a better representation than learning each alone. It is the principle behind instruction tuning and the foundation model, and its hazard is negative transfer, when unrelated tasks pull the shared weights in incompatible directions.
- `naive-bayes` Naive Bayes: A classifier that applies Bayes' theorem under an assumption it knows to be false, that the features are independent given the class, and works anyway. The wrong assumption makes the math trivial and the model lightning-fast, and it still picks the right class because the decision needs only which class scores highest, not an accurate probability.
- `online-learning` Online Learning: Learning from data that arrives in a never-ending stream, updating the model on each new example rather than training once over a fixed dataset. It suits any setting where data keeps coming and the world keeps changing, and its deep hazard is that absorbing the new tends to overwrite the old, the stability-plasticity dilemma.
- `self-supervised-learning` Self-Supervised Learning: The trick that made foundation models possible: manufacture supervision for free by hiding part of each example and training the model to predict it. No human labels, yet a concrete prediction target, which is why it, not labeled datasets, is what scaled, and predicting the hidden part well turns out to require genuinely understanding the data.
- `unsupervised-learning` Unsupervised Learning: Learning from data that carries no labels, where the goal is to uncover the structure already latent in the inputs: the groups, the directions of variation, the distribution that generated them. Its defining difficulty is the flip side of its freedom: with no answer key, there is often no way to say a result is right.

## Deep Learning Architectures (7)

- `backpropagation-through-time` Backpropagation Through Time: How a recurrent network is trained: unroll the loop across the time steps into one long feed-forward graph, then run ordinary backpropagation over it, so a gradient at the final step flows back through every earlier step. It is also where the vanishing and exploding gradient problems are born, because the same weights are multiplied through over and over.
- `batch-normalization` Batch Normalization: A layer that normalizes each feature's pre-activations across the current mini-batch to zero mean and unit variance, then rescales with learned parameters, which lets deep networks train faster and tolerate higher learning rates. Its defining quirk and weakness is a train/inference asymmetry: it uses live batch statistics while training and running averages at inference, so it falters when batches are tiny.
- `exploding-gradient` Exploding Gradient: The failure where gradients grow exponentially as they backpropagate through many layers or time steps, so a single update hurls the weights past anything useful and the loss spikes to infinity or NaN. It is the mirror image of the vanishing gradient, the same multiplicative dynamics running the other way, and gradient clipping is the direct fix.
- `hypernetwork` Hypernetwork: A neural network that generates the weights of another network rather than producing a prediction directly. Instead of learning the target weights, you learn a smaller network that outputs them, which lets one model adapt to many tasks or conditions by generating a different set of weights for each.
- `layer-normalization` Layer Normalization: Normalization that standardizes each example across its own features rather than across the batch, making it entirely batch-independent. That single change removes batch normalization's weaknesses, no train/inference gap, no collapse at batch size one, clean handling of variable-length sequences, which is why it is the normalization of the transformer.
- `weight-initialization` Weight Initialization: How a network's weights are set before training begins, a choice that matters far more than it looks. Too small and the signal vanishes through the layers; too large and it explodes; either way a deep network fails to train. Principled schemes, Xavier for symmetric activations and He for ReLU, set the random scale to preserve the signal's magnitude from layer to layer.
- `weight-normalization` Weight Normalization: A reparameterization that splits each weight vector into a direction and a separate length, so the optimizer tunes magnitude and orientation independently. It was proposed as a cheaper, batch-independent alternative to batch normalization, speeding optimization without depending on the statistics of a mini-batch.

## Deep Learning Theory (2)

- `benign-overfitting` Benign Overfitting: The surprising phenomenon where a model fits its training data perfectly, even memorizing noisy labels, and yet still generalizes well to new data, in flat contradiction of the classical warning that interpolating the training set must mean overfitting. It is one of the central puzzles that classical learning theory failed to predict about deep networks.
- `implicit-regularization` Implicit Regularization: The tendency of an optimization algorithm to prefer some solutions over others among the many that fit the data equally well, with no penalty added to the loss. It is the leading explanation for why overparameterized networks generalize: the dynamics, not the objective, quietly select good minima, which is also why optimizer, learning rate, batch size, and initialization are never neutral choices.

## Evaluation and Benchmarks (13)

- `benchmark` Benchmark: A fixed, shared task with a scoring rule, used to compare models on a capability and to track progress over time. Benchmarks make the field measurable, but a number is only as honest as the gap between the test and the real ability it stands in for.
- `benchmark-contamination` Benchmark Contamination: The leakage of benchmark test data into a model's training corpus, so high scores reflect memorization of the answers rather than genuine capability. Because models are pretrained on the same web that hosts almost every public benchmark, leakage is the default rather than the exception, which silently inflates reported results and gives every static evaluation a limited shelf life.
- `bertscore` BERTScore: A text-generation metric that scores a candidate against a reference by matching their contextual word embeddings rather than exact words, so a correct paraphrase with different vocabulary is credited where BLEU and ROUGE penalize it. It occupies a useful middle ground, fast and deterministic like n-gram overlap but meaning-aware, while still needing a reference and still foolable by a small meaning-flipping detail like a negation.
- `bleu-score` BLEU Score: An n-gram overlap metric for machine translation that scores a candidate against reference translations by modified n-gram precision, multiplied by a brevity penalty; higher is better, on a 0 to 1 scale. Its defining weakness is that it rewards surface form, not meaning: a correct paraphrase in different words scores poorly, which is exactly the gap semantic and judge-based metrics were built to close.
- `evaluation` Evaluation: The practice of measuring what a model can actually do, how well, and where it fails, through benchmarks, human judgment, model-graded scoring, and task-specific tests. It is the feedback signal the whole field steers by, and far harder than it looks because the thing being measured is open-ended.
- `hallucination` Hallucination: A fluent, confident model output that is factually wrong or unsupported by its source, arising because language models are trained to produce likely text rather than to track truth. It is not a bug bolted on but falls out of the objective: a fluent, well-formed falsehood is often more probable than an awkward admission of ignorance, and the model has no separate ledger of what it actually knows.
- `human-evaluation` Human Evaluation: The assessment of model outputs by people, the gold-standard signal for open-ended quality that automatic metrics only approximate; slow, costly, and subject to rater disagreement, but the ground truth against which cheaper proxies are validated. When a metric is said to "correlate with human preference," this is the thing it is correlated against, and its cost is the very bottleneck model judges were built to relieve.
- `humaneval` HumanEval: A benchmark of 164 hand-written Python programming problems that scores a model by whether the code it generates passes hidden unit tests, measuring functional correctness rather than text similarity, and reported as pass-at-k. Its departure from similarity metrics like BLEU is the whole point: BLEU rewards looking like a reference answer, HumanEval rewards being right.
- `lm-evaluation-harness` LM Evaluation Harness: A widely used open-source framework that standardizes how language models are evaluated across hundreds of benchmarks, fixing prompt formats, few-shot setups, and scoring so results are reproducible and comparable. Its lesson is that a benchmark name alone is not a measurement: the harness and its exact settings are part of the measurement, which is why two reported numbers for the same benchmark can disagree.
- `mmlu` MMLU: A multiple-choice benchmark of about 16,000 questions spanning 57 subjects, from elementary mathematics to professional law and medicine, used to measure the broad knowledge and reasoning of a language model with a single accuracy number. It captured breadth so a high score is hard to fake by overfitting one task, but because the questions are public and old, it is the motivating example of benchmark contamination: high scores can reflect having seen the answers.
- `pass-at-k` Pass at K: The standard metric for code generation: the probability that at least one of k sampled solutions to a problem passes its tests. Correctness is decided by running the code, an objective executable check rather than a judged or reference-matched one, which makes it far less gameable, and the k captures a property unique to generative models, their usefulness under sampling (coverage versus single-shot reliability).
- `perplexity` Perplexity: An intrinsic measure of how well a language model predicts a text, defined as the exponential of the average per-token cross-entropy; lower means the model is less surprised and assigns the data higher probability. It measures the pretraining objective directly and cheaply, but low perplexity does not guarantee usefulness: it rewards calibrated next-token probability, not correctness, helpfulness, or truth.
- `rouge` ROUGE: A family of recall-oriented overlap metrics for automatic summarization that scores a generated summary against references by shared n-grams or longest common subsequence; higher is better. It flips BLEU's emphasis from precision to recall because summarization's core risk is omission, but it shares the surface-overlap blind spot: it cannot credit a faithful paraphrase or penalize a fluent factual error.

## Foundations and History (4)

- `ai-winter` AI Winter: A collapse of funding and faith in AI, triggered when grand promises outrun what the technology can deliver. The field has frozen over at least twice, and the pattern, hype, disappointment, retreat, then a thaw driven by a genuinely new idea, is the single most useful thing to know when judging any AI claim, including today's.
- `dartmouth-workshop` Dartmouth Workshop: The 1956 summer meeting where AI got its name and its founding ambition. A small group proposed that a summer's work could make real progress on machines that think; the work took seventy years and counting, and the bet they made, on symbolic reasoning over perception, both shaped and misshaped the field for decades.
- `expert-system` Expert System: The commercial flagship of symbolic AI: a program that captured a human specialist's knowledge as thousands of if-then rules and applied them to give expert-level advice in a narrow domain. Its successes built an industry, and its one structural flaw, that experts cannot say what they know, collapsed it.
- `moravecs-paradox` Moravec's Paradox: The observation that the things humans find effortless, seeing, walking, grasping a cup, are the hardest to build into a machine, while the things we find hard, logic and arithmetic and chess, are the easiest. The difficulty hierarchy for machines is roughly the reverse of the one we feel, and the reason is written in evolutionary time.

## Frontier Architectures (7)

- `diffusion-language-model` Diffusion Language Model: A language model that generates text by iteratively denoising an entire sequence in parallel, the diffusion paradigm borrowed from image generation, rather than predicting one token at a time left to right. Its draw is the parallelism, refining many positions at once for large latency wins, but discrete text is harder to diffuse than continuous pixels, so autoregressive models still lead on quality.
- `joint-embedding-predictive-architecture` Joint Embedding Predictive Architecture: A self-supervised design that learns by predicting the representation of a masked or future part of an input from a visible part, working in an abstract embedding space rather than reconstructing raw pixels, so it captures structure without modeling unpredictable detail. Predicting in feature space invites representation collapse, which it prevents not with negatives but with architectural asymmetry, LeCun's bet for learning world-model-like structure.
- `linear-attention` Linear Attention: A reformulation of self-attention that removes the softmax so the computation reassociates: multiply keys by values first into a small running summary, then each query into that summary, cutting the cost from quadratic to linear and turning attention into a recurrence. Its honest difficulty is quality, a fixed-size state must compress an unbounded history, so it can miss the sharp recall softmax attention keeps.
- `neural-ode` Neural Ordinary Differential Equation: A model that replaces the discrete stacked layers of a network with a continuous transformation defined by a learned differential equation, so the hidden state evolves smoothly over a notional depth and is computed by a numerical ODE solver. Taking the limit of infinitely many infinitely small residual steps, it trains with constant memory via the adjoint method and lets the solver trade compute for accuracy at inference.
- `perceiver` Perceiver: A transformer-style architecture built to consume any kind of input, images, audio, point clouds, raw bytes, without modality-specific design, by funneling the huge input through a small fixed-size set of latent vectors. That funnel breaks attention's quadratic dependence on input size, letting one general architecture scale to very large or mixed inputs.
- `retentive-network` Retentive Network: A transformer-style architecture whose retention mechanism replaces self-attention with a decayed linear recurrence, distinctive for supporting three mathematically equivalent compute modes, parallel, recurrent, and chunkwise, so it trains like a transformer yet generates in constant memory. Its fixed decay is both strength and limit: clean and parallelizable, but content-blind where mamba adapts.
- `structured-state-space` Structured State Space: The structured state space sequence model (S4), the parameterization that turned deep state space models from a promising idea into a competitive architecture. Its breakthrough is the HiPPO initialization, a state matrix derived to optimally compress a continuous history, which lets the recurrence remember tens of thousands of steps where a random matrix would forget or blow up.

## Generative Models (7)

- `classifier-free-guidance` Classifier-Free Guidance: A sampling trick that makes a conditional diffusion model follow its condition more closely by extrapolating away from the model's own unconditional prediction, with no separate classifier needed. It is the near-universal dial for trading prompt fidelity against diversity in modern generators, and it costs a doubling of sampling compute since the network runs twice per step.
- `consistency-model` Consistency Model: A generative model trained so that every point along a diffusion trajectory maps directly back to the same clean origin, which lets it jump from noise to a finished sample in one evaluation instead of the dozens or hundreds a diffusion model needs. It attacks diffusion's central weakness, speed, by changing what the network predicts rather than by taking smaller steps.
- `flow-matching` Flow Matching: A generative framework that trains a network to predict a velocity field carrying samples from noise to data along continuous paths, the deterministic transport view of generation. By choosing simple, often straight paths, the learned trajectories integrate in far fewer steps than a diffusion model's, and a clever per-sample regression target makes the otherwise intractable field learnable.
- `generative-ai` Generative AI: The class of models that produce new content, text, images, audio, video, or code, by learning the distribution of their training data and sampling from it, rather than only classifying or predicting a label. It is the umbrella term for the technology behind the current wave of AI products.
- `text-to-image` Text-to-Image Generation: The task and model class that generate an image from a natural-language prompt, today a fairly settled stack: a text encoder, a latent diffusion model conditioned through cross-attention, and classifier-free guidance to set prompt adherence. The frontier has moved from raw photorealism to control, composing multiple objects correctly, rendering legible text, following long prompts faithfully.
- `video-generation` Video Generation: Synthesizing coherent video from a text prompt, an image, or another clip, extending image generation with the hard constraint that successive frames must agree across time. Temporal consistency, not single-frame quality, is what makes it difficult.
- `vq-vae` Vector-Quantized Variational Autoencoder: An autoencoder whose latent is discrete: it snaps each cell of an encoded image to the nearest entry in a learned codebook, so an image becomes a grid of integer tokens, a short sentence in a learned visual vocabulary. That discreteness is what lets the next-token machinery of language generate images, the discrete-token route to image synthesis that parallels continuous-latent diffusion.

## Geometry and Manifolds (1)

- `information-geometry` Information Geometry: The field that treats a family of probability distributions as points on a curved manifold, with the Fisher information as its natural metric. The reframing is powerful because it makes the difference between two distributions a geometric distance, which is what justifies the natural gradient and explains why ordinary parameter distance is the wrong ruler.

## Graph and Geometric Learning (1)

- `geometric-deep-learning` Geometric Deep Learning: A unifying framework that organizes deep learning architectures by the symmetries and geometric structure of their data, deriving each model from the principle that a network should respect the transformations under which its data's meaning is unchanged. It turns architecture design from art into something closer to derivation, and reveals grids, graphs, sequences, and manifolds, and the networks built for each, as one connected story.

## Industry and Markets (7)

- `ai-adoption` AI Adoption: The diffusion of AI capability into actual use across people, companies, and economies, which consistently lags the capability frontier. The binding constraint is rarely what the technology can do and almost always organizational: trust, workflow change, skills, and integration, so the gap between what is possible and what is deployed is the real story.
- `ai-agency-strategy` AI Agency Strategy: The business playbook, widely promoted online, of building a services company that uses AI to deliver work, automations, agents, content, for clients. A real opportunity wrapped in heavy hype, where the durable advantage is rarely the AI itself, which everyone can buy.
- `ai-operating-systems` AI Operating Systems: The framing of a coordinating AI agent as a personal or organizational operating system: an always-available layer that holds context, runs and schedules sub-agents, and mediates between a person and their tools and data. It is more a product vision and a piece of market rhetoric than a defined technical artifact.
- `deepseek` DeepSeek: A Chinese AI lab whose efficient, openly-released models, trained for a reported fraction of the usual cost, became a landmark in the open-versus-closed and efficiency-versus-scale debates. Its significance is less any single model than the proof that frontier-level capability did not obviously require frontier-level budgets.
- `enterprise-ai` Enterprise AI: The deployment of AI inside large organizations, where the hard part is rarely the model and almost always the surrounding context: messy proprietary data, security and compliance, integration with legacy systems, and the trust to put a probabilistic system into a business process. It is where the gap between an impressive demo and a dependable product is widest.
- `market-dynamics` Market Dynamics: The economic forces shaping the AI industry: who funds the enormous training runs, how labs compete and price, where value accrues across the stack, and whether the spending is justified by returns. The discourse theme that tracks AI as a business and a market, not only a technology.
- `open-models` Open Models: AI models whose weights are released for anyone to download, run, and modify, in contrast to closed models reachable only through a provider's API. The open-versus-closed split is one of the defining fault lines of the field, dividing it on safety, economics, and who controls the technology.

## Inference and Sampling (7)

- `logit-bias` Logit Bias: An inference control that adds a fixed offset to the logits of specified tokens before sampling, nudging the model toward or away from particular words or symbols with no retraining, used to ban tokens, steer vocabulary, or shape outputs. Its bluntness is its limit: the bias applies to a token in every context, and it works on tokens, not words or meanings.
- `prompt-caching` Prompt Caching: A serving optimization that stores the model's computed key-value state for a fixed prompt prefix so later requests sharing it skip recomputing, cutting both latency and cost for long, reused system prompts and documents. Because attention is causal, only an exact prefix match is reusable, which makes one discipline decisive: put the stable content at the very front and the variable query at the end.
- `prompt-engineering` Prompt Engineering: The craft of designing the text given to a model to reliably elicit the behavior you want, spanning phrasing, structure, examples, role-setting, and step-by-step instructions, the practical interface to a model whose weights you cannot change. Every technique is a way to shape the context so the model's most likely continuation is the one you want, and for agents it broadens into context engineering.
- `system-prompt` System Prompt: A high-priority instruction placed before the conversation that sets a model's role, rules, tone, and constraints for the whole session, the developer's standing brief, given more weight than ordinary user turns through a learned instruction hierarchy. That authority is real but soft, not a hard boundary, which is why a jailbreak or prompt injection in untrusted content can still talk the model out of it.
- `temperature` Temperature: A scalar that rescales a model's output logits before the softmax, controlling how random or deterministic the next-token choice is, low temperature sharpens toward the top token, high temperature flattens toward uniform. It is almost never used alone: truncate the garbage tail first with top-p or top-k, then let temperature reshape the survivors, because the order of those two operations decides whether high creativity stays coherent.
- `typical-sampling` Typical Sampling: A sampling method that keeps only tokens whose surprisal is close to the distribution's expected surprisal (its entropy), trimming both the over-probable and the very-improbable tail so generated text matches the information content the model expects, not just the highest-probability tokens. The premise is that natural human text conveys a fairly steady rate of information, neither the safest word nor noise.
- `verifier` Verifier: A model or procedure that judges whether a candidate output is correct, used to select among samples, filter training data, and supply the reward in RL on verifiable tasks. It rests on the generation-verification gap, that checking a solution is often far easier and more reliable than producing one, and the integrity of the verifier sets the ceiling on every method that leans on it.

## Learning Theory (4)

- `generalization-error` Generalization Error: The gap between how well a model performs on its training data and how well it performs on unseen data drawn from the same distribution, the quantity that actually matters since a model is deployed on data it has never seen. The entire point of learning is to minimize it, not the training error, which any flexible model can drive to zero.
- `pac-bayes` PAC-Bayes: A framework that bounds the generalization error of a distribution over hypotheses by how far that learned posterior strays, in KL divergence, from a prior fixed before seeing the data. Its KL penalty is a precise statement of Occam's razor, and by choosing the posterior cleverly it yields some of the only non-vacuous generalization bounds for deep networks.
- `pac-learning` PAC Learning: Leslie Valiant's framework defining what it means to learn from examples: with high probability, output a hypothesis whose error is small, using a sample that grows only polynomially in the relevant parameters. Both hedges, "probably" and "approximately," are unavoidable for a learner that sees only a finite random sample, and a class is learnable exactly when its VC dimension is finite.
- `rademacher-complexity` Rademacher Complexity: A data-dependent measure of a model class's capacity, defined by how well the class can fit pure random noise. It refines the cruder, distribution-free VC dimension by measuring complexity on the actual sample, and because bounding the weight norm bounds it, it is one of the few classical tools that says anything meaningful about over-parameterized networks.

## Mechanistic Interpretability (3)

- `data-attribution` Data Attribution: Tracing a model's behavior back to the specific training examples most responsible for it: which data points, if removed, would most change this prediction. It answers "why did the model do that" in terms of its data rather than its weights, which is the lever for debugging datasets, assigning credit or blame, and auditing what a model actually learned from.
- `linear-representation-hypothesis` Linear Representation Hypothesis: The hypothesis that neural networks encode high-level concepts as linear directions in activation space, so features can be read with linear probes, added and removed by vector arithmetic, and combined into the representations the model computes with. It is load-bearing: almost every practical interpretability method assumes some version of it, which means their successes are evidence for it and their failures mark its boundaries.
- `probing-classifier` Probing Classifier: A simple supervised classifier trained on a model's frozen internal activations to test whether a given piece of information is linearly decodable from them, used to map what a network represents at each layer. The linear probe is preferred precisely because it is weak: if a mere linear map can extract the property, the model did the representational work, but a probe shows only that information is decodable, not that the model uses it.

## NLP Foundations (7)

- `coreference-resolution` Coreference Resolution: The task of finding which expressions in a text refer to the same entity and grouping them into chains, linking "she" and "it" back to the right antecedents. It is a sensitive probe of genuine understanding because the hardest cases, the Winograd sentences where "it was too big" must bind to the trophy not the suitcase, turn on commonsense reasoning rather than grammar.
- `lemmatization` Lemmatization: The process of reducing an inflected word to its canonical dictionary form, its lemma, using vocabulary and grammatical analysis, so "ran" and "running" both become "run" and "better" becomes "good". Unlike stemming, it is linguistically informed: it always returns a real word and needs the part of speech to disambiguate ("saw" as verb to "see", as noun to "saw").
- `mathematical-reasoning` Mathematical Reasoning: The ability of a language model to solve problems that require multi-step deduction, symbolic manipulation, and exact answers, as in arithmetic, algebra, and competition mathematics. It is a sharp test of reasoning because math is unforgiving: one wrong step makes the final answer wrong, with nowhere to hide.
- `multi-turn-conversation` Multi-Turn Conversation: Holding a coherent exchange across many back-and-forth turns, where each reply must account for everything said before, not just the latest message. It is harder than single-turn answering because the model must track references, remember commitments, and stay consistent as the conversation grows.
- `n-gram-model` N-Gram Model: A statistical language model that estimates the next word from only the previous few, counting short word sequences in a corpus under a Markov approximation. Its defining flaw is a short, fixed memory, raising n to see further makes the counts explode and go sparse, and it is the direct conceptual ancestor of the modern LLM, a vastly more capable answer to the same question: what word comes next.
- `named-entity-recognition` Named-Entity Recognition: An information-extraction task that locates spans naming real-world entities, people, organizations, locations, dates, amounts, and labels each by type. It is the step that turns unstructured prose into structured, typed facts that search, question answering, and knowledge graphs build on, and its central difficulty is that the same surface form ("Washington") can be a person, place, or organization depending on context.
- `stemming` Stemming: A fast, rule-based normalization that strips suffixes to reduce words to a common stem, conflating related forms without consulting a dictionary, so the stem need not be a real word ("studies" to "studi"). Its crudeness produces two signature errors, overstemming (merging "universe" and "university") and understemming, and it sits at the lightweight end of a spectrum running to lemmatization.

## Optimization (3)

- `adam` Adam: The default optimizer of deep learning, combining momentum with a per-parameter learning rate by tracking running averages of both the gradient and its square. Dividing by the second of these nearly cancels each gradient's scale, so every parameter takes a similarly sized step, which is why Adam works with almost no tuning.
- `gradient-clipping` Gradient Clipping: A guardrail that caps the size of the gradient before each update, so one freakishly large gradient cannot blow the weights apart or poison the run with NaNs. It treats the symptom of instability, which is why it pairs with the structural fixes, good initialization, normalization, residuals, that address the cause.
- `lion-optimizer` Lion Optimizer: A memory-efficient optimizer, discovered by automated program search rather than designed by hand, that moves every parameter by a fixed amount in the direction of the sign of a momentum-smoothed gradient. Tracking one running average instead of Adam's two halves the optimizer memory, and its sign-only step is a rare case of machine search producing something simple enough to write down.

## Philosophy and AI, Advanced (7)

- `control-problem` Control Problem: The problem of how to ensure that an artificial intelligence far more capable than humans remains under meaningful human control and acts in accordance with human values, given that a sufficiently powerful misaligned system would be extremely difficult to constrain or stop. Capability control (boxing) is an unstable equilibrium, because a superhuman system out-thinks its containers and instrumental convergence pushes it to escape, which is why safety leans on motivation selection: making a strong optimizer we can still steer.
- `extended-mind` Extended Mind: The thesis, argued by Andy Clark and David Chalmers in 1998, that the mind is not confined to the brain but can literally extend into the tools and environment a person uses, when those external resources play the same functional role as internal cognition. Its engine is the parity principle: if a process inside the head would count as cognitive, the same process outside the head should too, which makes Otto's notebook part of his mind and the human-plus-tool the unit of intelligence.
- `intentional-stance` Intentional Stance: Daniel Dennett's account (1971, 1987) of how we predict and explain behavior by treating a system as a rational agent with beliefs and desires, a strategy whose success, not the system's inner stuff, is what makes it correct to call the system a believer. His deflationary claim is that being a believer just is being reliably predictable from the intentional stance: there is no further metaphysical fact of "real" intentionality a system must possess on top of being usefully described that way.
- `moral-patienthood` Moral Patienthood: The property of being an entity whose interests or welfare matter morally for its own sake, so that it can be wronged, distinguished from moral agency, the capacity to be held responsible for actions. Its leading criterion is sentience, the capacity to suffer, which collides with an epistemic problem: because subjective experience cannot be read off behavior or physiology with certainty, we may have to assign moral status under irreducible uncertainty.
- `panpsychism` Panpsychism: The view that consciousness or experience is a fundamental and ubiquitous feature of the physical world, present in some primitive form even in the smallest constituents of matter, rather than something that appears only in complex brains. Offered as a third way past physicalism's explanatory gap and dualism's interaction problem, its possibly-fatal objection is the combination problem: how countless micro-experiences could fuse into the single unified consciousness of a person.
- `phenomenology` Phenomenology: A philosophical tradition founded by Edmund Husserl in the early twentieth century that studies the structures of conscious experience from the first-person point of view, describing how the world is given to a subject rather than explaining it from the outside. Its enduring claim is that the first-person structure of experience is a legitimate and irreducible subject of rigorous study, not a vague residue to be explained away by the sciences of the brain.
- `superintelligence` Superintelligence: A hypothetical intellect that vastly exceeds the best human minds across virtually every domain, not merely faster but qualitatively better at science, strategy, and persuasion. The concept matters less as a prediction than as the setting in which the alignment and control problems become existential rather than merely practical.

## Philosophy of Mind and AI (10)

- `artificial-general-intelligence` Artificial General Intelligence: A hypothetical AI system with the broad, flexible, general-purpose cognitive ability of a human being, able to learn and reason across arbitrary domains rather than excelling at narrow tasks, and a focal point for debates about the nature and limits of machine mind. It has no agreed definition or test, and whether it is even possible turns on positions in the philosophy of mind, which is why it is the horizon where the conceptual entries in this category converge.
- `chinese-room` Chinese Room: A thought experiment by John Searle (1980) arguing that running the right program is not sufficient for genuine understanding, because a person manipulating symbols by rule can produce fluent answers in a language they do not understand at all. The room passes the behavioral test for understanding while no understanding is present anywhere in it, which is Searle's reductio: syntax, the manipulation of symbols by shape, does not by itself yield semantics.
- `frame-problem` Frame Problem: Originally a technical difficulty in AI (McCarthy and Hayes, 1969) about representing what stays unchanged when an action occurs, later generalized by philosophers into the puzzle of how a mind decides what is relevant without checking everything. The narrow version was tamed for circumscribed domains, but the difficulty migrated rather than vanished: relevance seems holistic, so no fixed local rule for ignoring things is guaranteed safe.
- `functionalism` Functionalism: The view in philosophy of mind that mental states are defined by their functional or causal roles, what they do and how they relate to inputs, outputs, and other states, rather than by the physical substrate that realizes them. Its engine is multiple realizability, the same state can run on neurons or silicon, which makes minds in principle substrate-independent and is why functionalism became the implicit philosophy of AI.
- `hard-problem-of-consciousness` Hard Problem of Consciousness: David Chalmers's name (1995) for the question of why and how physical processes in the brain give rise to subjective experience at all, distinguished from the easier problems of explaining the brain's functions and behaviors. It is different in kind, not degree: even a complete functional account of every discrimination, report, and behavior seems to leave open why any of it is accompanied by a felt inner quality rather than proceeding in the dark.
- `intentionality` Intentionality: The property of mental states being about, directed at, or representing something beyond themselves; revived as a mark of the mental by Franz Brentano (1874) and central to whether AI systems can be said to genuinely mean anything. The pivotal distinction for AI is original versus derived: a computer's symbols have at most derived aboutness, lent by the programmers and users who interpret them, never the original intentionality minds possess on their own account.
- `multiple-realizability` Multiple Realizability: The thesis that one and the same mental state can be realized by many different physical systems, so minds are not tied to a particular substrate, the argument that motivated functionalism and that underwrites the possibility of artificial minds on non-biological hardware. It was the decisive objection to the mind-brain identity theory, and it is the premise that makes "a machine could think" a coherent hypothesis rather than a category mistake.
- `philosophical-zombie` Philosophical Zombie: A hypothetical being physically and behaviorally identical to a conscious person but with no inner experience whatsoever; used by David Chalmers and others to argue that consciousness is not entailed by the physical facts. Its work is the conceivability argument: if a zombie is even conceivable, then consciousness could be absent while every physical fact stays the same, so consciousness is not reducible to the physical.
- `qualia` Qualia: The subjective, qualitative character of conscious experience, the felt "what it is like" of seeing red or tasting coffee, taken by many philosophers to be the aspect of mind that resists functional or physical explanation. They are the residue a complete objective, third-person account leaves out (Nagel's bat), which is what makes them the part of mind no behavioral or functional success seems able to reach.
- `symbol-grounding-problem` Symbol Grounding Problem: The problem, named by Stevan Harnad (1990), of how the symbols inside a formal system acquire meaning that is intrinsic to the system rather than parasitic on an outside interpreter, instead of being defined only in terms of other meaningless symbols. It is the constructive complement to the Chinese Room: where Searle argues that syntax does not suffice for semantics, Harnad asks what would, and answers grounding, anchoring some symbols directly in the system's own sensorimotor contact with the world.

## Probabilistic Machine Learning (4)

- `conformal-prediction` Conformal Prediction: A way to turn any model's raw output into a prediction set that is guaranteed, with a chosen probability, to contain the true answer, without assuming anything about the model or the data distribution beyond exchangeability. Instead of a single best guess, it returns a set whose size grows when the model is unsure, and that coverage guarantee is distribution-free and provable.
- `evidence-lower-bound` Evidence Lower Bound: A tractable lower bound on the log evidence of a probabilistic model. Its elegance is that one objective does double duty: raising it both fits the model to the data and drives an approximate posterior toward the true one, because the gap between the bound and the evidence is exactly the posterior's KL error. It is the training objective behind variational autoencoders and, in spirit, diffusion models.
- `importance-sampling` Importance Sampling: A Monte Carlo trick for estimating an expectation under a distribution you cannot sample, by drawing from an easier proposal distribution and reweighting each sample by how much the target favors it. It is unbiased no matter the proposal, but it lives or dies by that choice: a poor match makes a few samples dominate and the estimate useless.
- `uncertainty-estimation` Uncertainty Estimation: Getting a model to report not just an answer but how sure it is, and making that confidence trustworthy, so a system can defer, ask for help, or abstain when it does not know. It separates two distinct unknowns: irreducible noise in the data, and the model's own ignorance, which behave differently and call for different responses.

## Probability and Information Theory (2)

- `hypothesis-testing` Hypothesis Testing: A formal procedure for deciding whether an observed effect is real or could plausibly be chance: assume a null hypothesis of no effect, compute how surprising the data would be under it, and reject the null only if that surprise crosses a threshold. It is how a field decides whether a new model, drug, or change actually did something.
- `maximum-entropy-inference` Maximum-Entropy Inference: The principle that when choosing a probability distribution consistent with what you know, you should pick the one with the highest entropy, the most uncertain, least committal distribution that still fits the constraints. It is a formal rule against assuming more than your evidence justifies, and it derives many standard distributions and underlies maximum-entropy reinforcement learning.

## RAG, Embeddings, and Retrieval (7)

- `bm25` BM25: A classic lexical ranking function that scores a document from its overlapping query terms, weighting each by rarity, saturating the contribution of repeated occurrences, and correcting for document length. Those two corrections over tf-idf, saturation and length-normalization, are what make it the hard-to-beat baseline every new retrieval method is measured against, and its blind spot is vocabulary mismatch.
- `chunking` Chunking: The step in a retrieval pipeline that splits long documents into smaller passages before embedding, so each stored vector represents a focused unit that can be retrieved and cited on its own. It quietly sets a ceiling on everything downstream: a fact split across two chunks may never surface as a strong match, and an over-broad chunk poisons the context with noise, so bad boundaries are unrecoverable later.
- `hybrid-search` Hybrid Search: Retrieval that combines dense (semantic) and sparse (keyword) methods and fuses their results, so the system captures both meaning-based and exact-term matches that either alone would miss. Its rationale is that the two fail in opposite ways, and its engineering crux is fusion: because a cosine score and a BM25 score are not comparable, the standard answer (Reciprocal Rank Fusion) combines by rank position rather than raw score.
- `hyde` Hypothetical Document Embeddings: A retrieval technique that asks a language model to draft a hypothetical answer to the query, then embeds that draft instead of the query, so the search vector sits in the same register as real documents and matches them more closely. The fabricated specifics do not matter, because the draft is only a search probe never shown to the user; its job is to point the retriever at the right neighborhood.
- `knowledge-bases` Knowledge Bases: A curated, queryable store of an organization's or agent's knowledge, documents, facts, and structured records, built so a model can retrieve the right piece on demand rather than relying on what it memorized in training. The knowledge layer that grounds a model in specific, current, citable truth.
- `maximal-marginal-relevance` Maximal Marginal Relevance: A selection method that builds a result set one item at a time, each step choosing the candidate most relevant to the query while least redundant with what is already chosen, so the set is on-topic and diverse instead of a cluster of near-duplicates. A single relevance-vs-diversity knob tunes the balance, and in a RAG pipeline it stops the context window being wasted on three slices of the same source.
- `sparse-retrieval` Sparse Retrieval: Retrieval that represents queries and documents as high-dimensional sparse vectors over the vocabulary and ranks by term overlap, the classic keyword approach typified by BM25 and built on the inverted index. Exact lexical matching is genuinely the right tool for product codes, rare names, and error strings, the very queries dense retrieval handles worst, and its mirror weakness is blindness to synonyms.

## Reinforcement Learning (10)

- `advantage-function` Advantage Function: The quantity that measures how much better taking a particular action is than the state's average value, the action value minus the state value, the low-variance learning signal at the heart of modern policy optimization. Its power is isolation: it strips away how good the situation already was and leaves a signal centered on the action's own contribution.
- `behavior-cloning` Behavior Cloning: The simplest form of imitation learning, treating each expert state-action pair as a supervised example and training a policy to predict the expert's action, with no reward and no environment interaction. Its simplicity comes at a named cost, compounding errors from distribution shift: the moment the model strays from the demonstrated states, its mistakes snowball because nothing in the data shows how to recover.
- `counterfactual-learning` Counterfactual Learning: Learning what would have happened under a different choice from data about the choices that were actually made, so a system can evaluate or improve a policy without deploying it. It is how you answer "would this new recommender have done better" from logged history alone, and it lives or dies on correcting the bias of only ever seeing the actions that were taken.
- `generalized-advantage-estimation` Generalized Advantage Estimation: A method for estimating the advantage that blends temporal-difference errors across many horizons with an exponentially weighted average, exposing a single parameter that trades bias against variance and producing the smooth advantage signal PPO optimizes. One lambda knob slides between the biased one-step estimate and the high-variance Monte Carlo return, and the whole sum computes in one backward pass.
- `imitation-learning` Imitation Learning: Learning a policy by mimicking expert demonstrations rather than from a reward signal, used when a reward is hard to specify but examples of good behavior are easy to provide, with behavior cloning and inverse reinforcement learning its two main approaches. It sidesteps reward design entirely, and its classic weakness is distribution shift: an agent that only saw expert trajectories does not know how to recover once its own mistakes carry it off the path.
- `inverse-reinforcement-learning` Inverse Reinforcement Learning: Learning the reward function that explains an expert's behavior, rather than copying the behavior directly, on the premise that the reward is the most compact and transferable description of a task. It learns the why, not the what, so a recovered reward generalizes where a cloned policy is lost, and its central difficulty is that the problem is ill-posed: many rewards explain the same behavior.
- `policy-gradient` Policy Gradient: A family of methods that directly optimize a parameterized policy by following the gradient of expected reward, making high-reward actions more likely, rather than learning values and acting greedily. This direct route handles continuous and stochastic policies value-based methods struggle with, and its central problem is high variance, fixed by subtracting a value-function baseline, which leads straight to the advantage and actor-critic.
- `regret-minimization` Regret Minimization: A framework for learning under uncertainty that measures performance by regret, the gap between the reward you actually earned and the reward you would have earned with the best fixed choice in hindsight, and seeks algorithms that drive that gap's growth as slow as possible. It is the theoretical backbone of bandits, online learning, and the solvers that cracked poker.
- `reward-shaping` Reward Shaping: The practice of adding supplementary reward signals to guide an agent toward desired behavior, mostly to relieve sparse or delayed reward, most safely done with potential-based functions that speed learning without changing which policy is optimal. The guiding rule, shape the path without moving the destination, is exactly what the potential-based result guarantees and what reward hacking violates.
- `soft-actor-critic` Soft Actor-Critic: An off-policy actor-critic algorithm that maximizes reward plus policy entropy, encouraging the agent to act as randomly as possible while still succeeding, which yields strong exploration and stability and makes it a leading method for continuous control like robotics. The entropy term keeps exploration alive throughout training, and being off-policy (a replay buffer) makes it far more sample-efficient than on-policy PPO.

## Systems and Infrastructure (3)

- `compute-efficiency` Compute Efficiency: Getting more capability out of each unit of computation, through better architectures, training methods, and inference techniques, rather than only spending more. It is the counterweight to brute-force scaling, and the lever that decides whether AI's appetite for compute stays sustainable.
- `edge-deployment` Edge Deployment: Running a model on the device where the data is, a phone, a car, a camera, a sensor, instead of sending the data to a server. It trades the cloud's abundant compute for privacy, low latency, offline operation, and zero per-query cost, at the price of squeezing the model into a tight hardware budget.
- `local-inference` Local Inference: Running a model on your own hardware, a laptop, phone, or private server, rather than calling a provider's API. It trades the raw capability of the largest cloud models for privacy, control, offline use, and freedom from per-token fees, and is what quantization and small-model research exist to enable.

## Training and Fine-Tuning (6)

- `curriculum-learning` Curriculum Learning: A training strategy that orders examples from easy to hard rather than presenting them randomly, so a model builds competence gradually. Its payoff is clearest on hard optimization, rugged loss landscapes, reasoning, sparse-reward RL, where a random start rarely stumbles onto success and a curriculum supplies the intermediate footholds, though defining "difficulty" well is itself the hard part.
- `on-policy-distillation` On-Policy Distillation: Distilling a teacher model into a student by training on the student's own outputs, graded or corrected by the teacher, rather than on a fixed set of teacher-written examples. It closes the gap that ordinary distillation leaves, where a student trained only on the teacher's text never learns to recover from the mistakes it actually makes.
- `orpo` ORPO: A preference-alignment method that folds supervised fine-tuning and preference optimization into a single training stage by adding an odds-ratio penalty to the standard language-modeling loss, dispensing with both a reference model and a separate alignment phase. It is the most aggressive compression of the post-training recipe, aligning the model in the very same pass that teaches it format and behavior.
- `rejection-sampling-fine-tuning` Rejection Sampling Fine-Tuning: A post-training method that samples many candidate responses, keeps only the best by a reward model or verifier, and fine-tunes on that filtered set, then repeats, so a model learns from its own strongest outputs. It ratchets typical behavior upward toward occasional-best behavior while staying inside the stable machinery of supervised fine-tuning, no policy gradient or RL loop, and its ceiling is set by the scorer.
- `reward-modeling` Reward Modeling: The stage that distills human comparison judgments into a separate model predicting a scalar reward, so reinforcement learning can score thousands of fresh samples per step instead of querying humans. The whole alignment stage is bounded by its quality: a policy will find and exploit every flaw in the reward model, which is the mechanism behind reward hacking, and it is exactly the component direct preference methods remove.
- `rlvr` RLVR: Reinforcement learning where the reward comes from an automatic, objective check rather than a learned model of human preference: did the code pass the tests, did the math reach the right answer, did the proof verify. By replacing the gameable reward model with a ground-truth verifier, it sidesteps reward hacking and became the engine behind the recent leap in reasoning models.

## Transformers and LLMs (9)

- `attention-sink` Attention Sink: The empirical tendency of transformers to dump a large share of attention onto a few early tokens, often the very first, which act as a no-op drain the softmax demands. It is a clear case of an architectural constraint producing behavior unrelated to meaning, and its discovery fixed long-context streaming: keep a few "sink" tokens permanently in the cache or generation collapses.
- `byte-pair-encoding` Byte-Pair Encoding: The subword algorithm most LLMs use to build their tokenizer, born as a data-compression scheme and adapted to language: it grows a vocabulary by repeatedly merging the most frequent adjacent pair of symbols. Its byte-level variant runs over raw bytes, which guarantees any possible input encodes and removes the need for an unknown token entirely.
- `context-window` Context Window: The maximum number of tokens a language model can attend to at once, its working memory: everything it reasons over must fit inside or it is invisible. Quadratic attention is why the window is bounded and why longer is expensive, and even huge windows carry a soft caveat, models attend unevenly and recall the middle of a long context worse than the ends.
- `foundation-model` Foundation Model: A single large model pretrained on broad data at scale, usually by self-supervision, that serves as a general-purpose base adapted to many tasks by fine-tuning or prompting, rather than being built for one task. The name captures the shift to "pretrain once, adapt many", which concentrated capability in a few expensive bases, powerful but homogenizing, since their biases and blind spots propagate to everything built on them.
- `frontier-model` Frontier Model: The handful of most capable general-purpose models at any given moment, the ones pushing the boundary of what AI can do. It is a deliberately moving target rather than a fixed system, and the phrase carries real weight beyond marketing: "frontier AI" has become the policy label for models whose novel, not-yet-understood capabilities warrant special safety scrutiny.
- `kv-cache` KV Cache: An inference optimization that stores the key and value vectors of already-processed tokens so a generating transformer never recomputes them, turning each step from growing-with-length down to roughly constant. The price is memory: the cache grows linearly with the sequence and can exceed the model weights, which is the pressure behind multi-query and grouped-query attention.
- `long-context` Long Context: A model's ability to process very long inputs, from tens of thousands to millions of tokens, enabling whole-document and multi-document reasoning. It is hard for three converging reasons, quadratic attention, a ballooning KV cache, and scarce long-reasoning training data, and the honest catch is that the advertised window often exceeds the effective one, with recall sagging in the middle.
- `rotary-position-embedding` Rotary Position Embedding: The positional encoding in most current LLMs: rotate each query and key vector by an angle proportional to its position, so the dot product between them, and therefore the attention score, depends only on their relative distance. It uses no learned parameters and, because position enters as a rescalable rotation angle, extends to longer contexts gracefully.
- `sparse-attention` Sparse Attention: Attention computed over a chosen subset of token pairs instead of all of them, trading the transformer's full all-pairs interaction for a pattern that scales far better with sequence length. It rests on the observation that most of the attention in a trained model is local or structured, so the full quadratic computation is largely wasted.
